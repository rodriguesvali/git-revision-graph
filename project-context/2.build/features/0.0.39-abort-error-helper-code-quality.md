# 0.0.39 Abort Error Helper Code Quality

## Goal

Centralize common `AbortError` creation and detection across Git, graph layout, graph loading, render coordination, and Show Log request flows.

## Problem

Several runtime modules repeated the same `Error` naming convention and `instanceof Error && name === 'AbortError'` checks. The behavior was consistent, but the repeated implementation made cancellation handling noisier and easier to drift.

## Scope

- Added `src/errors.ts` with shared `createAbortError`, `throwIfAborted`, and `isAbortError` helpers.
- Reused the helpers from:
  - `src/gitExec.ts`;
  - `src/revisionGraph/layout/d3DagSugiyamaLayoutWorkerHost.ts`;
  - `src/revisionGraph/layout/layeredLayout.ts`;
  - `src/revisionGraph/panel/state.ts`;
  - `src/revisionGraph/backendServices/revisionLog.ts`;
  - `src/revisionGraph/backendServices/mergeAnalysis.ts`;
  - `src/revisionGraph/backendServices/snapshot.ts`;
  - `src/revisionGraph/renderCoordinator.ts`;
  - `src/showLog/loadRequests.ts`;
  - `src/showLogView.ts`.
- Preserved existing user-facing abort messages by passing the current message into the shared helper at each call site.
- Added direct helper coverage in `test/errors.test.ts`.

## Acceptance Criteria

- Runtime modules no longer duplicate AbortError construction or AbortError detection logic.
- Existing cancellation behavior and messages remain unchanged.
- Focused cancellation validation and full automated verification pass.
