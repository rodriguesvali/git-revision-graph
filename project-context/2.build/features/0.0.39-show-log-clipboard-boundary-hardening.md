# 0.0.39 Show Log Clipboard Boundary Hardening

## Goal

Finish the architectural bad-practices hardening work for review-panel clipboard boundaries by moving Show Log clipboard writes out of the editor panel provider.

## Problem

`ShowLogViewProvider` still combined selected file/commit lookup, copied text construction, and direct `vscode.env.clipboard` access for Show Log copy actions. That kept host-service access and action policy inside panel lifecycle code.

## Scope

- Added `src/showLog/clipboardActions.ts` as the owner for Show Log copy-to-clipboard actions.
- Routed `ShowLogViewProvider` through `copyShowLogChangeFileName`, `copyShowLogChangeFullPath`, and `copyShowLogCommitHash`.
- Preserved existing behavior for:
  - ignoring stale commit or changed-file selections;
  - copying rename-target file names and full paths;
  - copying only loaded commit hashes;
  - lazily loading VS Code clipboard services only when no explicit services are supplied.
- Added focused tests for file-name copy, full-path copy, commit-hash copy, and stale-selection ignores.

## Acceptance Criteria

- `ShowLogViewProvider` no longer touches `vscode.env.clipboard` for Show Log copy actions.
- Clipboard behavior is testable without loading VS Code extension-host APIs.
- Existing Show Log message dispatch, state lookup, file actions, and rendering behavior remain unchanged.
- Focused Show Log validation and full automated verification pass.

## Risks

- Manual Extension Development Host smoke testing should still confirm Show Log copy file name, copy full path, and copy commit hash actions write the expected values.
