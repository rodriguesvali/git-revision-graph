# 1.1.0 Toolbar Actions Flex Start

## Goal

Align the revision graph toolbar action group with a regular flex container that starts its contents from the left.

## Scope

- Change `.view-controls .toolbar-actions` from `inline-flex` to `flex`.
- Change the action group alignment from `justify-content: flex-end` to `justify-content: flex-start`.
- Remove the auto left margin that pushed the action group to the right.
- Add focused webview shell test coverage for the toolbar action layout rule.

## Non-Goals

- No change to toolbar buttons, commands, keyboard shortcuts, or webview messages.
- No change to graph rendering, repository selection, or Git workflows.

## Acceptance Criteria

- The toolbar action group uses `display: flex`.
- The toolbar action group uses `justify-content: flex-start`.
- The toolbar action group is not pushed right by `margin-left: auto`.

## Verification

- `npm run build` passed after the toolbar CSS update.
- `npm test` passed with 425 tests after adding webview shell coverage.
