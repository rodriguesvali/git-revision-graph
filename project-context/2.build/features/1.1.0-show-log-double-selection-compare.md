# 1.1.0 Show Log Double-Selection Compare

## Goal

Restore the Show Log commit context menu `Compare` action when exactly two commits are selected.

## Context

The Show Log multi-select cherry-pick flow changed the context menu for any multi-selected commit to only show `Cherry Pick`. That unintentionally hid the existing two-commit `Compare` action even though the compare message handling and Compare Results workflow still exist.

## Scope

- Show `Compare` and `Cherry Pick` when the context-clicked commit is part of exactly two selected commits.
- Keep `Cherry Pick` as the only action for multi-selections larger than two commits.
- Preserve single-commit actions, copy-reference-name behavior, and existing compare/cherry-pick message payloads.
- Update focused Show Log webview shell coverage.

## Non-Goals

- No changes to Compare Results presentation.
- No changes to cherry-pick ordering or conflict handling.
- No new commands, menus, settings, dependencies, or contribution points.

## Acceptance Criteria

- With exactly two selected Show Log commits, the commit context menu includes `Compare`.
- The same two-selected-commit context menu still includes `Cherry Pick`.
- With more than two selected commits, the context menu continues to offer only `Cherry Pick`.
- Focused Show Log webview coverage passes.
- Full automated verification passes before release handoff.

## Verification

- Focused Show Log webview shell test passed with `npm run clean:test && npm run build && tsc -p ./tsconfig.test.json && node --test out-test/test/showLogWebview.test.js`.
- `npm run build` passed through the focused Show Log webview validation.
- `npm test` passed with 415 tests after restoring double-selection compare.
- `git diff --check` passed after implementation and artifact updates.

## Manual Validation

- Open Show Log from a revision graph commit or range.
- Ctrl-click two commits, open the context menu on either selected commit, and confirm both `Compare` and `Cherry Pick` are present.
- Select three commits, open the context menu on a selected commit, and confirm only `Cherry Pick` is present.
