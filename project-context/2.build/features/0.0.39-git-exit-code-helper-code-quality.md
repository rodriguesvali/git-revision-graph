# 0.0.39 Git Exit Code Helper Code Quality

## Goal

Keep Git exit-code inspection in the shared Git-like error helper module.

## Problem

`isRefAncestorOfHead` manually inspected `error.code` to treat `git merge-base --is-ancestor` exit code `1` as a non-ancestor result. That local object-shape check duplicated the Git-like error handling concerns already owned by `src/errorDetail.ts`.

## Scope

- Added `hasGitExitCode` to `src/errorDetail.ts`.
- Reused the helper from `src/revisionGraph/repository/snapshot.ts`.
- Reused the same exit-code extraction inside `toErrorDetail`.
- Added direct `hasGitExitCode` coverage in `test/errorDetail.test.ts`.

## Acceptance Criteria

- Git exit-code matching is centralized in `src/errorDetail.ts`.
- `isRefAncestorOfHead` still treats merge-base exit code `1` as `false`.
- Focused error/backend validation and full automated verification pass.
