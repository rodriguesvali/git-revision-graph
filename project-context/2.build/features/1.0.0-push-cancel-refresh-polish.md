# 1.0.0 Push Cancel Refresh Polish

## Goal

Avoid reloading the revision graph when the current-branch push flow is canceled after the user chooses a push mode but before any push actually happens.

## Context

In VS Code, a normal push can surface a workbench permission/fork flow. If the user cancels that flow, the built-in Git API can return without throwing an error even though no remote update occurred. The extension previously treated that return as a successful push, showed success feedback, and scheduled a full graph reload.

## Scope

- Let the current-branch push adapter report whether a push actually happened.
- Cancel the prepared graph refresh when the adapter reports that no push happened.
- Keep successful push refresh behavior unchanged.
- Add regression coverage for the canceled/no-op push path.

## Non-Goals

- No change to the VS Code permission/fork dialog.
- No change to the push-mode QuickPick or force-push confirmations.
- No change to pull, sync, fetch, reload, tag push, or publish branch behavior.
- No new command, menu, view, setting, dependency, or contribution point.

## Acceptance Criteria

- If the normal push path returns without changing the current branch ahead state, the adapter reports no push.
- If no push happened, `pushCurrentBranchToUpstream` cancels the prepared refresh and returns `false`.
- No graph reload or success message is emitted for a canceled push.
- Successful push paths still refresh the graph.

## Risks

- The normal push adapter now uses a targeted `git rev-list --left-right --count HEAD...@{upstream}` check around `repository.push(...)`. If that ahead check is unavailable, the adapter preserves previous behavior and treats the returned push as successful.

## Verification

- Focused ref action validation passed with 84 tests after adding the canceled push refresh guard. This includes `npm run build` and test compilation through the focused command.
- `npm test` passed with 395 tests after the canceled push refresh guard. This includes `npm run build` through the test script.
- `git diff --check` passed after the canceled push refresh guard.
