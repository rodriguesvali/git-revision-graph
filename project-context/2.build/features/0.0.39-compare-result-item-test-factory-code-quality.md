# 0.0.39 Compare Result Item Test Factory Code Quality

## Goal

Reduce duplicated Compare Results test setup by sharing a focused `CompareResultItem` factory.

## Problem

Compare Results action tests recreated the same `CompareResultItem` object shape locally. The repeated setup made tests longer than the behavior they exercise and made future changes to the item contract noisier.

## Scope

- Added `createCompareResultItem` to `test/fakes.ts`.
- Reused the shared factory from Compare Results item action and restore action tests.
- Removed local `CompareResultItem` fixture builders from those tests.
- Kept scenario-specific refs, labels, and changes explicit at the call sites.

## Acceptance Criteria

- Compare Results action tests no longer repeat the full `CompareResultItem` object shape for common fixtures.
- Existing assertions around item opening, worktree comparison fallback, restore confirmation, and restore execution remain unchanged.
- Focused Compare Results validation and full automated verification pass.
