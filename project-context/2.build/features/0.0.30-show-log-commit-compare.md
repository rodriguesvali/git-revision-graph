# Show Log Commit Compare

## Build Status

Build implementation complete for the `0.0.30` Show Log commit compare workflow.

Version bump, package generation, and Marketplace publishing remain unapproved.

## Goal

Let users compare two commits directly from Show Log by selecting the first commit, Ctrl-selecting the second commit, and choosing `Compare` from the commit context menu.

## Implementation Summary

- Replaced the single local Show Log selected commit with an ordered, two-commit local selection.
- Preserved normal click behavior: selects one commit and expands or collapses details.
- Added Ctrl-click behavior: updates comparison selection without expanding or collapsing details.
- Added a `Compare` commit context-menu action that appears only when two selected commits include the context-menu commit.
- Added `Compare with Worktree` for the single selected commit context-menu state.
- Added bounded `compareCommits` webview message validation.
- Added bounded `compareCommitWithWorktree` webview message validation.
- Injected `CompareResultsPresenter` into `ShowLogViewProvider`.
- Added testable commit-compare helpers that call `repository.diffBetween` or `repository.diffWith` and open the existing Compare Results view.

## Acceptance Criteria

- Normal click still selects and toggles commit details.
- Ctrl-click adds a second commit to the comparison selection without toggling details.
- A third Ctrl-click keeps only the previous latest selection and the new commit.
- `Compare` is shown only when two commits are selected and the menu opens over either selected commit.
- `Compare with Worktree` is shown when exactly one commit is selected and the menu opens over that commit.
- Compare Results opens with the first selected commit as base and the second selected commit as compare.
- Compare Results opens against the worktree for single-commit worktree comparisons.
- Missing or unloaded commit hashes are ignored.
- Empty diffs show an informational message instead of opening Compare Results.

- Open Show Log.
- Click one commit and confirm details still expand or collapse.
- Ctrl-click another commit and confirm details are not toggled.
- Right-click the second selected commit and confirm `Compare` appears.
