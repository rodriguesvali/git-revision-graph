# Show Log hover contrast

## Goal

Make hover states in the Show Log webview noticeably easier to perceive, especially on dark themes where the previous hover color was too close to the page background.

## Scope

- Strengthen commit-row hover feedback with a more opaque themed hover background.
- Add a subtle focus-border-colored leading accent and outline on hovered commit rows.
- Align Show Log context-menu items with the revision graph menu style through the shared `context-menu-item` class.
- Preserve VS Code theme tokens and avoid hard-coded theme-specific backgrounds.

## Acceptance criteria

- Hovering a Show Log commit row is visually distinct from neighboring rows.
- Expanded commit rows still remain identifiable while gaining hover feedback.
- Context-menu hover/focus states use the same clean rounded filled highlight as the revision graph menu.
- Automated build and Show Log webview tests pass.

## Risks and notes

- Visual contrast still depends on installed VS Code theme tokens, but the fallback path now mixes in focus-border accents for a stronger cue.
- Manual Extension Development Host validation remains required for final subjective contrast review.
