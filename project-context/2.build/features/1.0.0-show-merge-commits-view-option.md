# 1.0.0 Show Merge Commits View Option

## Goal

Make merge commits optional in the revision graph so the default graph stays visually cleaner while users can still inspect merge commits when needed.

## Context

The graph currently renders unreferenced merge commits as regular visible cards. In repositories with many merge commits, this can create a noisy visual presentation and make the graph harder to scan.

## Scope

- Add a `Show Merge Commits` option to the existing `View` menu.
- Keep the option unchecked by default.
- Apply the option through the revision graph projection state.
- Hide unreferenced merge commits by default while preserving graph ancestry through edge paths.
- Preserve referenced merge commits, roots, forks, and unreferenced tips needed to keep graph navigation meaningful.
- Keep the setting within the current webview state contract and validation path.

## Non-Goals

- No redesign of the graph layout algorithm.
- No change to Show Log merge-commit rendering.
- No persistence change beyond the existing graph view state flow.
- No package, Marketplace, or command contribution changes.

## Acceptance Criteria

- The `View` menu includes a `Show Merge Commits` checkbox.
- The checkbox is unchecked when a graph view starts with default projection options.
- With the option off, unreferenced merge commits are omitted from graph nodes.
- With the option on, merge commits are restored to the projected graph.
- Edges across hidden merge commits preserve the relevant ancestry path information.
- The toggle participates in toolbar busy state and graph reload feedback like the other view options.

## Risks

- Hiding merge commits could make some branch relationships feel less explicit even though edge paths preserve the underlying ancestry.
- Merge commits with visible refs still appear because hiding those would also hide user-facing labels.

## Verification

- Focused graph, webview shell, and message-validation tests passed after adding the option.
- Full automated verification is recorded in `project-context/3.deliver/release.md`.
