# 0.0.18 Refresh Efficiency and Activity Bar Icon

## Goal
Improve the next release with a narrow pair of changes: reduce unnecessary revision graph reloads and make the Activity Bar icon visually stronger beside native VS Code icons.

## User Workflow
- Users can change graph view options, checkout references, or trigger repository state changes without paying for a full graph snapshot reload when the existing snapshot is still valid.
- Users can identify `GIT Revision Graph` more easily in the Activity Bar.

## Scope
- Add an explicit graph snapshot reload gate so the controller reuses the current snapshot for projection rebuilds and metadata-only repository events.
- Fall back to a full rebuild when metadata patching cannot represent the new repository state.
- Keep manual refresh, fetch, and graph-changing Git actions able to force a full rebuild.
- Scale the Activity Bar SVG artwork by about 30% inside its existing 24px icon box.

## Non-Goals
- No version bump or Marketplace publishing in this cycle.
- No redesign of graph rendering, data limits, or Git command strategy.
- No new persistent cache layer beyond the current in-memory snapshot.

## Touched Surfaces
- `src/revisionGraph/controller.ts`
- `src/revisionGraph/panel/state.ts`
- `src/revisionGraphRefresh.ts`
- `media/icon-source.svg`

## Acceptance Criteria
- Projection-only graph changes reuse the current snapshot when it is clean.
- Repository state and checkout events attempt metadata patching first and avoid full reloads when possible.
- Explicit full rebuild requests still reload the graph snapshot.
- The icon appears visually larger while preserving the same contribution paths.
- `npm run build` and `npm test` pass.
