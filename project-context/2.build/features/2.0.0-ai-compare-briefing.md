# 2.0.0 AI Compare Briefing

Status: Implemented; automated verification current, manual Extension Development Host smoke pending

## Goal

Let a user explicitly request an AI-generated briefing for the active Compare Results review without
leaving the existing editor panel. The briefing should summarize the change, identify review and risk
areas, and suggest verification work while keeping Git facts and all mutations deterministic.

## User workflow

1. Open a ref-to-ref or ref-to-worktree comparison.
2. Select `AI Briefing` in Compare Results.
3. Review the generated plain-text briefing in its bounded, vertically scrollable panel above the file
   list, and copy it to the system clipboard when useful.
4. Close and reopen the briefing locally, or regenerate it when desired; continue using the existing
   file and unified-diff actions.

## Scope

- Use the stable VS Code Language Model API available in the pinned `@types/vscode` `1.90.0`.
- Select an available Copilot model only after the user invokes the webview action.
- Build context from the active comparison labels, safe changed-file inventory, and a bounded unified
  diff.
- Exclude known sensitive paths such as `.env`, credentials, private keys, certificates, and package
  registry credential files before Git loads the AI diff.
- Display model output as plain text; never interpret it as HTML.
- Present generation as a compact AI-icon action with a tooltip and accessible label, and let the user
  dismiss or reopen the generated panel without another model request.
- Bound the briefing body to `min(45vh, 420px)` with local vertical scrolling, keeping its title and
  header actions visible.
- Expose an icon-only `Copy to clipboard` action only for ready results; the webview emits intent only,
  while the extension host copies its current trusted briefing state.
- Cancel active work when the comparison, repository, panel, or request changes, and reject stale
  completions.
- Keep the model dependency optional so all existing workflows remain available without Copilot.

## Non-goals

- No automatic review decision, merge, restore, commit, push, or other Git mutation.
- No background generation, persistent prompt/diff/output storage, telemetry, chat participant, or
  model-provider authentication.
- No claim that AI output is authoritative or that arbitrary secrets inside otherwise safe source files
  can be detected perfectly.
- No new Marketplace command, view, setting, runtime dependency, or minimum VS Code version.

## Acceptance criteria

- Completed Compare Results states expose an accessible `AI Briefing` action only when an AI
  generator is configured.
- Loading and ready states are visible in the retained Compare Results panel; regeneration replaces the
  previous result.
- The briefing panel has an accessible close action; the icon action reopens a dismissed ready result
  without invoking the model and otherwise generates or regenerates according to current state.
- Ready briefings can be copied through a validated no-payload webview message; idle/loading states
  cannot copy stale or missing content.
- Empty/model-unavailable/content-excluded cases produce concise information messages and leave the
  review usable.
- Unexpected failures use the shared modal error presenter.
- Ref-to-ref and ref-to-worktree prompts contain only normalized, non-sensitive paths and bounded diff
  text.
- A stale or canceled response cannot overwrite a newer comparison or briefing.
- Message validation, dispatch, state shaping, prompt policy, webview rendering, and presenter
  orchestration have deterministic tests.

## Architecture

- `src/compareResults/aiBriefing.ts`: pure path policy, prompt/context shaping, result contracts.
- `src/workbenchAiCompareBriefing.ts`: VS Code Language Model adapter and token-budget enforcement.
- `src/compareResults/briefingWorkflow.ts`: bounded diff loading for the active compare state.
- `src/compareResults/briefingController.ts`: request lifecycle, cancellation, unavailable/error
  presentation, and stale-result guard.
- `src/compareResultsView.ts`: provider composition and serializable-state publication.
- `src/compareResults/webviewBriefing.ts`: accessible action, plain-text result panel, and scoped styles.
- Existing Compare Results message validation/dispatch and serializable webview state remain the browser
  boundary.
- Existing document backend remains the Git text-data owner and gains optional path/signal bounds.

## External contract evidence

- Context7 library: `/websites/code_visualstudio_api`.
- Current documentation confirms model selection may return no models, requests require user action and
  consent, and failures include permission, quota, and disappearing-model cases.
- Current documentation now describes a newer response stream shape, so exact implementation
  compatibility is taken from the pinned `@types/vscode` `1.90.0`, where
  `LanguageModelChatResponse.text` is the supported async text stream.

## Risks and mitigations

- Sensitive content: filter paths before invoking Git and cap file/diff context.
- Prompt size: use model token counting and reduce diff content before sending.
- Nondeterminism: test prompt construction and orchestration with fakes; do not call live models in tests.
- Stale state: bind each request to the exact Compare Results state and cancellation token.
- Large diffs: bound captured Git output and clearly report when no safe context remains.
