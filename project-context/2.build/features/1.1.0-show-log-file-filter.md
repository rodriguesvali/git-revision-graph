# Show Log File Filter

## Goal

Add an inline file search box to expanded Show Log commits so users can quickly find changed artifacts in commits with large file counts.

## User Workflow

- Open Show Log for a ref or range.
- Expand a commit with changed files.
- Type in `Filter files...` above the file list to narrow visible file rows by path or status.
- Clear the filter with the inline clear button.

## Scope

- Add a client-side, per-expanded-commit file filter in the Show Log webview.
- Match the Compare Results file filter visual treatment and placeholder.
- Keep file context menus, double-click compare, and keyboard actions working on visible rows.

## Non-goals

- Do not change commit-level filtering or backend log queries.
- Do not add multi-commit file search.
- Do not change Show Log message contracts.

## Touched Surfaces

- `src/showLogWebview.ts`
- `test/showLogWebview.test.ts`

## Acceptance Criteria

- Expanded commits with changed files show a `Filter files...` search box above the file rows.
- The file filter aligns to the right side of the expanded file area, matching the Compare Results visual convention.
- Filtering narrows the expanded file list by file path or status.
- A no-match state is shown when no files match the active filter.
- The graph continuation column remains visually continuous across the filter row and filtered results.
- The filter is local to the expanded commit and does not trigger host requests.
- Existing Show Log file actions and commit actions remain unchanged.
