# 1.0.0 Show Log Copy Reference Name

Build implementation complete for the `1.0.0` Show Log copy reference name polish.

## Goal

Allow users to copy reference names directly from Show Log commits that have loaded references.

## Scope

- Add a Show Log commit context-menu action only when the selected commit has one or more refs.
- Copy the actual reference `name` value, not the decorated display label.
- Preserve existing Show Log compare, commit detail, reset, tooltip, and changed-file actions.

## UX Decision

Commits with one ref show a direct `Copy Reference Name` action. Commits with multiple refs show a `Copy Reference Name` submenu listing each available reference name.

## Acceptance Criteria

- Commits without refs do not show the copy-reference action.
- Commits with a single ref copy that ref name from the commit context menu.
- Commits with multiple refs expose a submenu where each ref name can be copied individually.
- Webview messages are validated and the host copies only references that belong to a currently loaded Show Log commit.

## Verification

- Passed: `npm run build`.
- Passed: focused Show Log clipboard, message handler, webview shell, and webview message validation tests.
- Passed: `npm test` with 399 tests.
- Passed: `git diff --check`.

## Risk

- Small menu-density risk for commits with many refs. The action is hidden for unreferenced commits and grouped into a submenu when needed.
