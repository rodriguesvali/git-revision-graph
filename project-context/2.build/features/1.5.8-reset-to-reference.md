# Reset to graph reference

## Goal

Expose `Reset to this` from graph references so users can reset the current local branch to the commit behind a visible non-HEAD target.

## Scope

- Add `Reset to this` to the revision graph context menu for resettable graph targets.
- Keep `HEAD` and the current branch alias out of the reset action to avoid redundant/no-op destructive actions.
- Exclude stash targets because they are not branch reset targets.
- Send the selected target commit hash through the webview message; the reference only provides the discoverable menu affordance and label.
- Reuse the existing `resetCurrentBranchToCommit` workflow, confirmation, clean-worktree guard, mutation coordination, and refresh behavior.

## Acceptance criteria

- A visible tag, remote branch, local branch, or unreferenced commit can expose `Reset to this`.
- The reset workflow receives the selected commit hash, not the reference name.
- The webview boundary validates and authorizes reset messages against the current graph state.
- `HEAD`, current branch alias, and stash targets do not expose/authorize reset.
- Automated build and tests pass.

## Risks and notes

- This is a destructive `git reset --hard` path; the action stays under the destructive section and keeps the existing confirmation dialog.
- Manual Extension Development Host verification is still required to confirm menu placement and wording in the live webview.
