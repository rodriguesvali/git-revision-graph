# Load-Only Revision Graph Refresh

## Goal

Remove the incremental `Update Revision Graph` refresh paths and route revision graph refreshes through the full `Load Revision Graph` rebuild path.

## Rationale

Maintainer feedback identified that `Update Revision Graph` can feel slower than `Load Revision Graph` because metadata patches may still reload refs, rebuild layouts, recalculate merge-blocked targets, run fingerprint checks, and force webview rendering. The new behavior favors one predictable refresh algorithm over multiple optimized paths with uneven perceived latency.

## Scope

- Keep `full-rebuild` as the only `RevisionGraphRefreshIntent`.
- Route repository state changes, checkouts, projection option changes, sync, push, conflict, reset, merge, branch, tag, fetch, and manual reload flows through `full-rebuild`.
- Remove host/webview `patch-metadata` and `patch-workspace-state` message paths.
- Remove metadata patch state helpers and tests.
- Prepare follow-up repository event suppression before explicit Git mutations that will request a full graph load.
- Keep prepared follow-up event suppression active for the whole suppression window so repeated `state` or `checkout` events from the same operation do not schedule extra graph loads.
- Preserve graph snapshot loading, repository overlays, render cancellation, multi-repository handling, and existing Git action workflows.

## Acceptance Criteria

- Every graph refresh uses the `Loading revision graph...` label.
- Webview state updates use complete `update-state` payloads rather than incremental patch messages.
- Push-only sync, workspace reset, merge abort, and conflict follow-up refreshes rebuild the graph fully.
- Explicit graph-action refreshes prepare repository event suppression before mutating Git state.
- Repeated repository events during the prepared suppression window do not schedule additional loads for the same operation.
- `npm run build` passes.
- `npm test` passes.

## Risks

- More frequent `git log` execution may cost more on large repositories when only workspace metadata changed.
- Selection and viewport preservation from metadata patches is removed for those refreshes.
- Manual Extension Development Host validation should confirm reload, repository switching, scope changes, push-only sync, reset workspace, abort merge, and conflict state updates feel acceptable with full rebuilds.
