# Performance Review

## Scope
- Start the graph-load performance review with a focused webview metadata patch optimization.
- Keep graph data contracts, VS Code contribution points, backend loading, and release metadata unchanged.

## Change
- Add a render key to revision graph node markup.
- During metadata patches, skip DOM replacement for graph nodes whose rendered content, class, dimensions, refs, title, and summary have not changed.
- Continue refreshing graph caches after metadata patches, while rebinding delegated node-layer handlers only when nodes were actually replaced.
- Replace `Array.shift()` queue traversal with cursor-based BFS in graph highlight distance and commit graph ancestor/descendant helpers.
- Use viewport-only minimap sync after metadata patches when no graph node was replaced, keeping full minimap redraws for changed nodes.
- Replace per-row crossed-gap arrays in dynamic vertical gap calculation with compact per-source ranges before applying fan-out clearance.
- Extend opt-in graph load tracing to include host-to-webview state delivery and webview apply durations for `init-state`, `update-state`, metadata patches, and workspace-state patches.
- Build a single ref-to-commit index while resolving merge-blocked targets so large visible ref sets do not rescan the full commit graph for every reference.

## Acceptance Criteria
- Metadata patches still avoid full scene and edge rerenders.
- Unchanged node cards are not replaced during metadata patches.
- Changed node cards are still replaced and keep their previous horizontal drag offset.
- BFS helpers avoid per-item array shifting while preserving traversal behavior.
- Metadata patches without node replacements avoid full minimap content redraws.
- Large fan-out corridor layout avoids building duplicate crossed-gap arrays while preserving dynamic clearance behavior.
- `gitRevisionGraph.traceLoading` captures webview apply timings without adding trace payloads when tracing is disabled.
- Merge-blocked target calculation preserves graph-derived blocking behavior while avoiding per-reference graph scans.
- Existing graph, webview, metadata patch, and shell tests remain green.

## Verification
- `npm run build` passed.
- `npm test` passed with 299 tests.

## Follow-Up
- Profile a real large repository with `gitRevisionGraph.traceLoading` enabled to identify the next highest-cost phase after metadata patching, BFS traversal, minimap sync, vertical gap calculation, webview apply tracing, and merge-blocked target indexing.
