# 1.5.4 Focus Selected Range

## Status

Implemented.

## Objective

Add one user-visible graph navigation feature for `1.5.4`: let users focus the revision graph on the path between two selected revisions without opening a secondary panel.

## Scope

- Add `Focus Range` to the two-revision graph context menu.
- Store the active range as a revision graph projection option.
- Reproject the current graph snapshot when applying or clearing the range filter.
- Show a removable range filter chip in the graph toolbar.
- Clear the active range when the user changes the main graph scope.

## Implementation Notes

- Added `revisionRange` to `RevisionGraphProjectionOptions`.
- Projection resolves range endpoints by hash or visible ref name and keeps the base and compare commits visible while collapsing intermediate commits into the existing `through` edge path.
- Webview messaging reuses `set-projection-options`; clearing the range sends `revisionRange: null` through the validation boundary.
- Snapshot reuse ignores `revisionRange` because the range is a projection over the loaded graph data.
- UX review replaced the generic `Range:` filter text with a distinct active-state chip: a range icon, `Focus` caption, typographic direction arrow, full-label tooltip, and an explicit `Exit Focus Range` action.
- The active range group now exposes both endpoints through a dynamic accessible label; truncated visual labels retain their full value in the tooltip.
- The two-selection context action is state-aware: it reads `Focus Range` without an active interval, changes to `Update Focus Range` for different ordered endpoints, and is omitted when the selected base/compare pair already matches the active interval.
- The active Focus Range indicator is placed after the graph action groups, at the end of the toolbar.

## Manual Validation Focus

- Ctrl/Cmd-click two visible revisions and right-click one selected revision.
- Choose `Focus Range` and confirm the graph narrows to the selected path.
- Confirm the toolbar shows the active range and its clear button restores the previous graph projection.
- Change the main graph scope and confirm the range filter is cleared.
