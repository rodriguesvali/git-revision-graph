# 1.2.0 Copy Hash Menu Polish

## Status

Implemented for the `1.2.0` cycle.

## Objective

Make commit hash and reference-name copy actions easier to discover and label them consistently across the revision graph and Show Log context menus.

## Scope

- Add `Copy Hash` to the Show Log single-commit context menu.
- Rename the revision graph context menu label from `Copy Commit Hash` to `Copy Hash`.
- Rename reference-name copy labels to `Copy Ref Name` in the revision graph and Show Log context menus.
- Reuse the existing host message contracts and clipboard handlers.
- Do not add new commands, contribution points, settings, or clipboard behavior.

## Implementation Notes

- Show Log commit context menus now expose `Copy Hash` only for the single-commit context to avoid ambiguity during compare and multi-selection contexts.
- The Show Log menu action posts the existing `copyCommitHash` webview message for the commit that opened the context menu.
- The revision graph menu continues posting the existing `copy-commit-hash` message; only the visible menu label changed.
- Existing ref-name copy messages are preserved; only the visible labels changed.

## Manual Validation Focus

- Open Show Log, right-click a single commit, and confirm `Copy Hash` is available.
- Select two or more Show Log commits, right-click a selected commit, and confirm `Copy Hash` is not offered.
- Confirm `Copy Hash` copies the clicked commit hash in the single-commit context.
- Confirm Show Log and revision graph reference-name copy actions show `Copy Ref Name`.
- Right-click a revision graph commit/ref and confirm the graph menu label is `Copy Hash`.

## Rollback

Remove the Show Log context-menu item and restore the longer hash/ref-name labels if the shorter labels cause confusion.
