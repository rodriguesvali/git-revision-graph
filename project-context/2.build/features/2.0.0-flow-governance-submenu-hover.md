# 2.0.0 Flow Governance Submenu Hover

Status: Complete
Last updated: 2026-07-16
Target version: `2.0.0`

## Goal

Keep Flow Governance submenu actions reachable while the pointer moves from the parent context-menu
item into the fixed-position submenu.

## User Workflow

1. Open the revision graph context menu on a governed reference.
2. Hover `Flow Governance` to open its submenu.
3. Move the pointer into a submenu action and select it without the submenu disappearing.

## Scope And Acceptance Criteria

- Remove the six-pixel dead zone between parent menu and submenu in both right- and left-opening
  placements by using a one-pixel overlap.
- Delay pointer-driven submenu closure briefly and cancel it when the pointer re-enters the menu.
- Cancel pending closure when the context menu closes or keyboard focus reopens the submenu.
- Preserve immediate keyboard navigation, Escape/ArrowLeft handling, click dispatch, viewport
  clamping, accessibility attributes, and the existing Flow Governance action set.
- Add deterministic placement tests and runtime contract checks for delayed/cancelled closing.

## Non-goals

- No change to Flow Governance policy, Git actions, host/webview protocol, menu labels, commands,
  settings, dependencies, or Marketplace metadata.

## Risks And Verification

- The delay must be short enough to keep menu dismissal responsive while allowing diagonal pointer
  movement across borders and display scaling.
- Run the focused revision graph webview tests, `npm run quality:check`, `npm run build`, `npm test`,
  `git diff --check`, and `graphify update .`.
- Manual Extension Development Host smoke remains required for right- and left-opening submenus,
  slow/diagonal pointer movement, keyboard navigation, and submenu action dispatch.

## Release Impact

- User-visible bug fix under the existing Flow Governance context menu; no packaging or publication
  action is authorized by this artifact.
