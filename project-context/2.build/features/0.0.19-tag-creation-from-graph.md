# 0.0.19 Tag Creation From Graph

## Goal

Add local Git tag creation from the Revision Graph context menu for graph-selected references and visible commits.

## User Workflow

- A user opens the `GIT Revision Graph` view.
- The user right-clicks a visible branch, tag, remote ref, stash-excluded commit target, or unreferenced commit.
- The context menu includes `Create Tag`.
- The extension prompts for a tag name.
- The extension creates a lightweight local tag at the selected revision and refreshes the graph so the new tag is visible when tag display is enabled.

## Scope

- Add a graph context menu action for tag creation.
- Resolve the target revision from the same graph target model used by branch creation.
- Prompt for a tag name using native VS Code UI.
- Create a lightweight local tag at the selected revision.
- Refresh the graph after successful tag creation.
- Show success, cancellation, and error feedback through existing workbench UI patterns.
- Preserve multi-repository behavior by using the graph's current repository.

## Non-Goals

- No package version bump until explicitly approved.
- No Marketplace packaging or publishing until explicitly approved.
- No remote tag push workflow in this initial slice.
- No annotated tag message workflow in this initial slice.
- No tag rename flow.
- No broad redesign of graph rendering, context menus, or repository refresh architecture.

## Acceptance Criteria

- Users can create a local tag from a visible branch, tag, remote ref, or unreferenced commit in the Revision Graph.
- Canceled prompts do not create tags or show misleading success feedback.
- Duplicate or invalid tag names surface a clear error message from Git or validation.
- Successful tag creation refreshes the graph and shows an information message.
- Existing branch creation, checkout, merge, delete, compare, and log actions continue to work.
- `README.md` no longer lists tag creation as not included once the feature is implemented.
- `npm run build` and `npm test` pass.
