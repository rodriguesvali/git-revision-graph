# 0.0.39 Show Log File Action Cohesion

## Goal

Reduce architectural coupling in the Show Log review panel by moving file-action helpers out of `ShowLogViewProvider` and into a focused Show Log module.

## Scope

- Added `src/showLog/fileActions.ts` for Show Log file diff actions, file path selection, file name selection, and root-commit parent fallback.
- Routed `ShowLogViewProvider` file diff and clipboard actions through the new helper module while keeping panel lifecycle and selection lookup in the provider.
- Preserved existing behavior for:
  - comparing a file change against the first parent commit;
  - falling back to the empty tree hash for root commits or missing entries;
  - comparing a file change against the worktree;
  - using rename targets for copied file names and full paths when available.
- Added focused tests for parent hash fallback, rename path selection, and injected diff service calls.

## Acceptance Criteria

- Show Log file actions keep the same command-facing behavior and message contracts.
- `ShowLogViewProvider` no longer owns file action path/parent-hash/diff details.
- File action helpers are testable without loading the VS Code extension host APIs.
- Focused Show Log validation and full automated verification pass.

## Verification

- `npm run clean:test && npm run build && tsc -p ./tsconfig.test.json && node --test out-test/test/showLogFileActions.test.js out-test/test/showLogStateLookup.test.js out-test/test/showLogMessageHandler.test.js out-test/test/showLogShared.test.js out-test/test/showLogWebview.test.js` passed with 20 tests.

## Risks

- File diff actions depend on the same parent-ref ordering as the provider-local implementation.
- Clipboard path behavior depends on preserving the existing rename-target preference.
- The default diff services are loaded lazily so pure helper tests do not require the `vscode` module.
