# 1.2.0 Legacy Visibility Context Removal

## Goal

Remove obsolete activation work that resets the removed side-bar graph visibility context.

## Context

The current product exposes one Source Control-launched editor graph panel. The old side-bar graph and Source Control companion graph are no longer contributed in `package.json`.

The `gitRefs.revisionGraphVisible` context key remained only as legacy code:

- `src/extension.ts` awaited a helper during activation;
- `src/viewLayout.ts` executed `setContext` with `false`;
- `src/revisionGraphTypes.ts` exported the context key;
- `test/viewLayout.test.ts` tested that isolated reset.

No active manifest `when` clause or runtime workflow consumed the context value.

## Scope

- Remove the activation-time visibility initialization.
- Remove the unused context constant.
- Delete the obsolete `viewLayout` module.
- Delete its dedicated test.
- Update `1.2.0` release artifacts.

## Non-Goals

- No changes to contributed commands, menus, editor panels, or activation events.
- No removal of the runtime compatibility command `gitRefs.openRevisionGraph`.
- No graph controller, Git workflow, webview, or repository-selection changes.
- No package cleanup, bundling, VSIX packaging, or Marketplace publication.

- `src/extension.ts`

## Acceptance Criteria

- Activation does not execute `setContext` for `gitRefs.revisionGraphVisible`.
- Production and test source contain no references to the removed context or helper.
- The Source Control command and editor graph contribution remain unchanged.
- `npm run build`, `npm test`, and `git diff --check` pass.

## Test Plan

- Run `npm run build`.

- Invoke `View Git Revision Graph` from Source Control and confirm the editor graph opens normally.

## Risks

- A third-party extension or user keybinding could theoretically inspect the undocumented legacy context. The key is internal, no longer controls a contributed surface, and is removed as dead compatibility state.
