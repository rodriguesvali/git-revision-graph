# 0.0.37 Central Review Panels

## Goal

Move `Compare Results` and `Show Log` out of left-side Activity Bar review views and into editor-area panels, matching the central placement of the main revision graph.

## Scope

- Convert Compare Results from `WebviewViewProvider` to a singleton `WebviewPanel`.
- Convert Show Log from `WebviewViewProvider` to a singleton `WebviewPanel`.
- Remove Compare Results and Show Log Activity Bar view/container contributions from `package.json`.
- Preserve existing compare, worktree compare, Show Log, commit details, reset, restore, clipboard, and diff message handling.
- Close dependent review panels when the main revision graph panel closes.
- Update README, manifest coverage, and release readiness notes.

## Non-Goals

- Redesigning Compare Results or Show Log content layout.
- Merging Compare Results or Show Log into the main graph panel.
- Changing Git operations or compare/log semantics.
- Changing minimap, graph layout, or toolbar behavior.

## Acceptance Criteria

- Compare Results opens in the editor area when a compare produces results.
- Show Log opens in the editor area from graph context-menu actions.
- No `Git Revision Graph - Compare` or `Git Revision Graph - Show Logs` Activity Bar containers are contributed.
- Closing the main graph panel disposes Compare Results and Show Log panels.
- Existing automated manifest, webview, compare, and Show Log workflow tests pass.
