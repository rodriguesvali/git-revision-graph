# Abort Conflicted Merge

## Build Status

Build implementation complete for the `0.0.28` merge recovery slice.

Update on 2026-05-02: tightened graph-toolbar visibility so `Abort Merge` requires both unresolved merge changes and a repository `MERGE_HEAD`. Other unresolved conflict states such as rebase/cherry-pick conflicts should keep showing Source Control conflict status without exposing merge abort. The webview stylesheet now explicitly hides toolbar buttons with the `hidden` attribute so the abort action is not merely disabled in clean workspaces, and the conflict LED now shares the abort button border color with a brighter red hover state. Merge conflicts and successful merge aborts now apply lightweight workspace-state patches so the dirty LED and abort action update without reloading or rerendering the graph before a merge commit exists.

Version bump, package generation, and Marketplace publishing remain unapproved.

## Goal

Let users safely abandon a conflicted merge from the revision graph when conflict resolution is blocked or needs outside support.

## User Workflow

- A user starts a merge from the revision graph.
- Git leaves the repository with unresolved merge conflicts.
- The graph toolbar shows an `Abort Merge` action while the conflicted merge is still in progress.
- The user confirms the destructive action.
- The extension runs `git merge --abort`, refreshes the graph, and reports the outcome.

## Scope

- Add an `Abort Merge` toolbar action only for conflicted merge state.
- Gate the graph toolbar action on `MERGE_HEAD` as well as unresolved merge changes.
- Patch only workspace state after a conflicted merge or successful merge abort instead of refreshing graph history.
- Add a validated `abort-merge` webview message.
- Add a testable `abortCurrentMerge` ref action with modal confirmation.
- Execute `git merge --abort` through the existing `ReferenceManager` Git CLI adapter.
- Refresh the graph with a full rebuild after a successful abort.
- Open Source Control if aborting fails so the user can inspect the remaining conflict state.

## Non-Goals

- Use `git reset --hard` as the default recovery path.
- Provide advanced conflict guidance or automated conflict resolution.
- Abort non-merge operations such as rebase or cherry-pick.

## Acceptance Criteria

- The graph shows `Abort Merge` only while unresolved merge changes are present and the repository has `MERGE_HEAD`.
- `Abort Merge` requires confirmation before running Git.
- Successful abort runs `git merge --abort`, performs a full graph refresh, and shows a success message.
- Failed abort shows an error and opens Source Control.
- Webview messages cannot invoke abort merge unless the current graph state has a conflicted merge.
