# Show Log File Revert

## Goal

Allow users to restore a workspace file from the version shown in an expanded Show Log commit.

## User Workflow

- Open Show Log for a ref or range.
- Expand a commit and open a changed file context menu.
- Choose `Revert to this`.
- If the workspace copy already has local changes, confirm before it is overwritten or deleted.

## Scope

- Add `Revert to this` to expanded Show Log file context menus.
- Route the webview action through the Show Log message validation and handler.
- Restore the workspace file from the selected commit version.
- Reuse the existing worktree restore guard so local workspace changes trigger confirmation.

## Non-goals

- Do not reset the whole commit or branch.
- Do not alter commit context-menu `Reset to this` behavior.
- Do not add bulk file restore from Show Log.

## Acceptance Criteria

- Expanded Show Log file rows expose `Revert to this` in their context menu.
- The action replaces the workspace copy with the selected commit copy.
- Deleted files in the selected commit are removed from the workspace.
- Locally changed workspace copies require confirmation before restore.
- Clean workspace copies restore without an extra prompt.
- Malformed webview messages are rejected.
