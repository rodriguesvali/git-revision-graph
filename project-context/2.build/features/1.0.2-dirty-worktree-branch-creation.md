# 1.0.2 Dirty Worktree Branch Creation

## Goal

Let users create and check out a new branch from the revision graph when the workspace has ordinary uncommitted changes, matching VS Code Source Control behavior.

## Context

The branch creation workflow reused the generic workspace mutation guard, so it blocked whenever the worktree had tracked or untracked changes. VS Code Source Control allows creating a new branch and checking it out with non-conflicting worktree changes, which supports the common flow of starting work first and naming the branch afterward.

## Scope

- Allow `Create New Branch` to continue when the workspace has ordinary worktree changes.
- Keep unresolved conflict/merge guards before branch creation.
- Preserve clean-worktree requirements for checkout, merge, pull, sync, reset, cherry-pick, and destructive flows.
- Preserve existing branch-name validation, remote tracking setup, existing-branch handling, and user feedback.

## Non-Goals

- No redesign of branch checkout prompts.
- No change to direct checkout of an existing branch or reference.
- No new commands, settings, dependencies, or contribution points.

## Acceptance Criteria

- Creating and checking out a new branch from a dirty but non-conflicted workspace no longer shows the clean-worktree warning.
- The new branch is created with checkout enabled through the VS Code Git API.
- Branch creation remains blocked when merge conflicts are unresolved.
- Focused ref action coverage and full automated verification pass before release handoff.

## Verification

- `npm run build` passed through the focused ref action validation.

## Manual Validation

- Modify a file in a Git workspace without committing.
- Open `View Git Revision Graph` from Source Control.
- Use `Create New Branch` from the current branch or current commit and enter a new branch name.
- Confirm the branch is created and checked out while the workspace changes remain present.
- Repeat with unresolved conflicts and confirm the extension opens Source Control and blocks branch creation.
