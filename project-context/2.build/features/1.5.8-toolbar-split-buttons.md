# Toolbar split buttons

## Goal

Make advanced toolbar commands discoverable without relying on a hidden 500ms long-press gesture.

## Scope

- Replace the hidden long-press affordance on `Push` and `Reload` with explicit split-button menu affordances.
- Keep the primary `Push` and `Reload` buttons on their existing default actions.
- Put extra actions behind adjacent chevron buttons:
  - Push menu: `Push with Force With Lease`, `Push with Force`.
  - Reload menu: `With Empty Cache`.
- Preserve webview accessibility affordances with `aria-haspopup`, `aria-expanded`, and menu roles.

## Acceptance criteria

- Primary `Push` still runs the normal current-HEAD push flow.
- Primary `Reload` still reloads the graph normally.
- Chevron buttons clearly expose extra actions and can be toggled open/closed.
- Toolbar busy/disabled state disables both the primary and menu halves consistently.
- Automated build and webview tests pass.

## Risks and notes

- Force push commands remain visually discoverable but still require opening the menu, reducing accidental activation risk.
- Manual Extension Development Host verification is still required to confirm final visual spacing in VS Code themes.
