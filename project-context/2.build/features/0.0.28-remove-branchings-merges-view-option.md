# Remove Branchings And Merges View Option

## Build Status

Build implementation complete for the `0.0.28` graph view simplification slice.

## Goal

Remove the low-value `Show Branchings & Merges` option from the revision graph View menu and from the projection contract so users stay on the stable refs-focused graph.

## Rationale

- The option exposes internal topology detail that is not central to the primary user workflow of browsing refs and running ref actions.
- The option changes the revision graph `git log` command by adding sparse branch/merge topology loading, increasing behavior variance across large repositories.
- A real Extension Development Host report showed a blank graph after enabling the option in `Current Branch` scope.

## Scope

- Remove the webview checkbox and client-side listeners for `Show Branchings & Merges`.
- Remove `showBranchingsAndMerges` from projection options, validation, refresh invalidation, and default state.
- Stop adding `--sparse` to revision graph `git log` args.
- Keep refs-only structural connector behavior that preserves readable lines between visible refs.

## Non-Goals

- Remove merge commits from Git history or Show Log.
- Change merge, checkout, compare, or branch workflows.
- Add a replacement advanced topology mode.
