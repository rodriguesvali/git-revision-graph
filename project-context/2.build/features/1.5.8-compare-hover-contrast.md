# Compare Results hover contrast

## Goal

Apply the approved Show Log hover affordance pattern to the Compare Results webview so file rows and menus are easier to perceive on dark themes.

## Scope

- Strengthen Compare Results file-row hover feedback with a themed background.
- Add a subtle focus-border-colored leading accent and internal outline on hovered/focused rows.
- Preserve selected-row state while making selected-row hover distinct.
- Align Compare Results context-menu items with the revision graph menu style through the shared `context-menu-item` class.
- Preserve VS Code theme tokens and avoid hard-coded theme-specific backgrounds.

## Acceptance criteria

- Hovering a Compare Results file row is visually distinct from neighboring rows.
- Selected rows keep their selection state while showing hover feedback.
- Compare Results context-menu hover/focus states match the clean rounded filled menu highlight used by the revision graph and Show Log.
- Automated build and Compare Results webview tests pass.

## Risks and notes

- Final perceived contrast still depends on the active VS Code theme, so manual Extension Development Host validation remains useful.
