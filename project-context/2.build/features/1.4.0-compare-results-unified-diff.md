# 1.4.0 Compare Results Unified Diff

## Goal

Add a `Unified Diff` action to the `Compare Results` editor panel so users can generate the full unified diff for the active ref-to-ref or ref-to-worktree comparison without returning to the revision graph.

## Context

The revision graph already supports unified diff generation for two selected revisions through `openUnifiedDiffDocument`. The Compare Results panel currently provides a file-level review queue, double-click file diffs, clipboard actions, compare-with-worktree, and restore flows, but it does not expose the full textual diff for the comparison session.

## Scope

- Add a toolbar button labeled `Unified Diff` to the Compare Results editor panel.
- Show the button when the active Compare Results state is a ref-to-ref or ref-to-worktree comparison.
- Route the webview action through the existing Compare Results message validation and dispatcher.
- Reuse the existing revision graph document backend and diff editor flow.
- Generate worktree output from the selected ref through staged and unstaged changes, then append patches for untracked files listed by the active comparison.
- Add focused regression coverage for webview state, webview shell output, message validation, message dispatch, Git execution, and the document backend.

## Non-Goals

- No new command palette command or manifest contribution.
- No separate persistent unified diff panel.
- No changes to graph-side unified diff behavior.

## Acceptance Criteria

- Ref-to-ref Compare Results sessions expose a `Unified Diff` button.
- Ref-to-worktree Compare Results sessions expose the same button; empty state does not.
- Clicking the button posts a validated `unifiedDiff` message and opens the unified diff for the active ref pair or ref-to-worktree session.
- Worktree unified output includes tracked staged/unstaged changes and active untracked files, while preserving bounded Git execution and repository-relative path validation.
- Existing file-level compare, compare-with-worktree, restore, copy, filter, selection, and context-menu behavior remain intact.
- `npm run build`, `npm test`, and `git diff --check` pass.

## Risks

- `git diff <ref>` does not include untracked files by itself, so those files require bounded `git diff --no-index` patches and explicit handling of its expected exit code `1`.
- The Compare Results provider now depends on the document backend; keep the dependency narrow and reuse the existing backend instance from extension activation.
- Toolbar layout should remain compact on narrow editor widths.
