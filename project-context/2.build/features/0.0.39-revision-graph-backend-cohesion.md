# Revision Graph Backend Cohesion

## Goal

Continue the `0.0.39` architectural cohesion work by narrowing the broad revision graph backend surface and moving Show Log history loading into a dedicated backend service while preserving the existing public backend facade.

## Scope

- Add `src/revisionGraph/backendServices/revisionLog.ts` as the owner of Show Log revision history loading, filtered log scanning, and commit change loading.
- Keep `DefaultRevisionGraphBackend` as the stable facade returned by `createRevisionGraphBackend()`.
- Introduce narrower backend contracts for graph snapshots, graph state, document loading, revision log loading, merge analysis, and Show Log changes.
- Update graph state building to depend on `RevisionGraphStateBackend` instead of the full backend.
- Update diff/detail document helpers to depend on `RevisionGraphDocumentBackend` instead of the full backend.
- Update Show Log to depend only on revision log, document, and Show Log change ports.
- Simplify graph state tests so their fakes implement only snapshot and merge-analysis methods.

## Acceptance Criteria

- `createRevisionGraphBackend()` keeps returning a backend compatible with the existing controller, editor panel, and Show Log wiring.
- Graph state loading only requires snapshot loading and merge-blocked target analysis.
- Show Log history loading, filtered pagination, and commit change loading remain behavior-preserving.
- Unified diff and commit details helpers only require document-loading methods.
- Existing imports from `src/revisionGraph/backend.ts` remain valid.
- `npm run build` passes.
- `npm test` passes.
- `git diff --check` passes.

## Verification

- `npm run build` passed after extracting revision log loading and narrowing backend contracts.
- `npm test` passed with 312 tests after extracting revision log loading and narrowing backend contracts. This includes `npm run build` through the test script.
- `git diff --check` passed after extracting revision log loading, narrowing backend contracts, and updating verification artifacts.

## Risks

- Backend port extraction can drift runtime wiring if `createRevisionGraphBackend()` stops satisfying controller or Show Log needs.
- Show Log filtering can regress if scan limits, default branch scope, ref-kind parsing, or abort handling change during extraction.
- Graph state tests can become too narrow if the state backend contract omits a method that state loading genuinely requires.
