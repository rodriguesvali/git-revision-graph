# 0.0.32 Editor Panel Reload Button

## Goal

Restore an explicit `Reload` action inside the revision graph editor panel toolbar.

## User Workflow

A user opens `View Git Revision Graph` from Source Control. Because the graph now runs as an editor `WebviewPanel`, the previous external view-title refresh action is no longer available. The user should be able to reload the current graph directly from the graph toolbar.

## Scope

- Add a `Reload` button to the graph webview toolbar.
- Reuse the existing `refresh` webview message and controller `full-rebuild` path.
- Keep the button disabled while another toolbar action is busy.
- Update README and webview tests for the restored action.

## Non-Goals

- Reintroducing the old primary Activity Bar graph view.
- Reintroducing external `view/title` graph commands for removed graph views.
- Changing refresh intent classification or refresh performance behavior.
- Changing fetch behavior.

## Touched Surfaces

- `src/revisionGraphWebview.ts`
- `src/revisionGraph/webview/script/bootstrap.ts`
- `src/revisionGraph/webview/script/interactions.ts`
- `test/revisionGraphWebview.test.ts`
- `README.md`

## Acceptance Criteria

- The editor graph toolbar shows `Reload`.
- Clicking `Reload` posts the existing `{ type: 'refresh' }` message.
- The controller handles the action through the existing full rebuild refresh path.
- The button participates in toolbar busy/disabled state.
- Tests lock the button markup and message wiring.
