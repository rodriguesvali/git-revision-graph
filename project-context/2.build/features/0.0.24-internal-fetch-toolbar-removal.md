# 0.0.24 Internal Fetch Toolbar Removal

## Goal
Remove the redundant `Fetch` control from the revision graph webview toolbar while preserving the native VS Code view title action.

## Scope
- Remove the internal webview `Fetch` button from the graph action toolbar.
- Remove webview runtime wiring that only existed for the internal button.
- Keep `gitRefs.fetchCurrentRepository` contributed in the external view toolbar.
- Keep the controller fetch workflow unchanged for the external command.

## Acceptance Criteria
- The revision graph webview no longer renders `fetchButton`.
- The webview runtime no longer registers a `fetchButton` click listener.
- `Fetch Current Repository` remains available from the VS Code view title toolbar.
- Webview shell tests cover the removed internal control.

## Implementation Notes
- The webview shell now keeps `workspaceLed`, `Reorganize`, and zoom actions in the internal toolbar.
- The command contribution in `package.json` remains unchanged because it is the desired external action.

## Verification
- `npm run build`
- `tsc -p ./tsconfig.test.json`
- `node --test out-test/test/revisionGraphWebview.test.js`
- `npm test`

## Manual Validation
- Launch the Extension Development Host.
- Open the GIT Revision Graph view.
- Confirm the internal toolbar no longer shows `Fetch`.
- Confirm the external view title toolbar still exposes `Fetch Current Repository`.
