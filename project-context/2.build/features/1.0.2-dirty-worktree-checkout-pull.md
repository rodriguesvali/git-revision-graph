# 1.0.2 Dirty Worktree Checkout And Pull

## Goal

Let checkout and pull follow VS Code Source Control behavior by attempting the Git operation when the workspace has ordinary uncommitted changes.

## Context

The extension used a generic clean-worktree guard before checkout and pull. VS Code Source Control allows these operations with non-conflicting local changes and lets Git report a failure only when the operation would overwrite or conflict with those changes.

## Scope

- Allow checkout of another reference with ordinary workspace changes.
- Allow current-branch pull with ordinary workspace changes.
- Keep unresolved conflict guards before checkout and pull.
- Preserve existing user feedback, refresh behavior, and Git API usage.

## Non-Goals

- No change to reset, merge, cherry-pick, or destructive flows.
- No new commands, settings, dependencies, or contribution points.
- No custom conflict resolution flow beyond opening Source Control for unresolved conflicts.

## Acceptance Criteria

- Checkout no longer shows a clean-worktree warning for ordinary local changes.
- Pull no longer shows a clean-worktree warning for ordinary local changes.
- Checkout and pull remain blocked while conflicts are unresolved.
- Focused ref action coverage and full automated verification pass before release handoff.

## Verification

- Focused ref action tests passed with `npm run clean:test && npm run build && tsc -p ./tsconfig.test.json && node --test out-test/test/refActions.test.js`.
- `npm run build` passed through the focused ref action validation.
- `npm test` passed with 419 tests after allowing dirty-worktree checkout and pull.
- `git diff --check` passed after implementation and artifact updates.

## Manual Validation

- Modify a file in a Git workspace without committing.
- From the revision graph, check out another branch/reference and confirm the extension attempts the checkout.
- From the current branch actions, run pull and confirm the extension attempts the pull.
- Repeat both operations with unresolved conflicts and confirm the extension opens Source Control and blocks the operation.
