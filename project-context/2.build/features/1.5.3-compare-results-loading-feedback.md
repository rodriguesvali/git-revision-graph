# 1.5.3 Compare Results Loading Feedback

## Goal

Remove apparent freezes in Compare Results when compare and `Unified Diff` operations involve large outputs.

## Release Placement

The maintainer confirmed on 2026-06-28 that `1.5.2` has already been published. This fix is therefore assigned to the next patch release, `1.5.3`. Package metadata remains `1.5.2` until a separate version-bump approval is given.

## Scope

- Add immediate visual feedback after the `Unified Diff` toolbar action is clicked.
- Disable the `Unified Diff` action while generation is already in flight.
- Present unified diff failures that block command completion in a modal dialog instead of a toaster.
- Open Compare Results immediately in a `Loading results...` dialog state while ref-to-ref or ref-to-worktree compare results are being loaded.
- Restore the previous Compare Results state, or close the panel when there was no previous state, if compare loading ends with no differences or an error.
- Keep the existing ref-to-ref and ref-to-worktree unified diff generation paths unchanged.
- Preserve Compare Results filters, selection, and list state while showing transient progress.

## Non-Goals

- No change to Git diff command arguments or diff document content.
- No new contribution points, commands, settings, dependencies, packaging, or publication.
- No attempt to stream partial unified diff output.

## Acceptance Criteria

- Clicking `Unified Diff` changes the action text to `Generating Diff...` immediately.
- Repeated clicks are ignored while the unified diff is being prepared.
- The action returns to `Unified Diff` after success, cancellation, stale worktree refresh, or error handling.
- Unified diff generation errors, including output limit failures such as `33554432 bytes`, are shown as modal error dialogs.
- Starting a compare opens Compare Results with a centered `Loading results...` dialog before changed-file results are available.
- Compare errors that block result loading are shown as modal error dialogs.
- Existing Compare Results file actions, filtering, selection, and contextual actions remain unchanged.
