# Git-Aware Row Monotonicity

## Goal

Prevent projected revision graph descendants from rendering below their visible parents when a merge or simplified edge initially anchors the descendant from a deeper ancestor.

## Scope

- Keep the existing Git-aware layout strategy and cache orchestration.
- Add a topological row-order pass after initial row assignment.
- Preserve side-branch sibling layering where descendants of the same visible parent align with their mainline sibling.
- Namespace the layout cache as `git-aware-v3` so persisted positions from the earlier row strategy are not reused.

## Acceptance

- For every projected edge `child -> parent`, the child renders above the parent.
- Shared structural ancestors such as an unlabeled `94...` connector cannot appear above one visible child and below another.
- Existing Git-aware mainline and side-branch placement tests remain green.
