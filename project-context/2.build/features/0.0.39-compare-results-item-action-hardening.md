# 0.0.39 Compare Results Item Action Hardening

## Goal

Continue the architectural bad-practices hardening work by moving Compare Results item-action policy out of the editor panel provider.

## Problem

`CompareResultsViewProvider` still owned item-level action details such as selecting the correct diff refs, choosing the worktree comparison fallback ref, and formatting clipboard file name/path text. That mixed panel lifecycle and workbench orchestration with item action policy, making the provider harder to reason about and leaving these rules less directly testable.

## Scope

- Added `src/compareResults/itemActions.ts` for Compare Results item diff and clipboard helpers.
- Routed provider item open, compare-with-worktree, copy file name, and copy full path actions through the new helper module.
- Preserved existing behavior for:
  - between-ref diffs;
  - worktree diffs;
  - worktree comparison fallback order: `worktreeRef`, `rightRef`, `leftRef`;
  - rename-target file names and full paths;
  - multi-item clipboard order.
- Added focused tests for diff service calls, fallback refs, rename-target paths, and clipboard list formatting.

## Acceptance Criteria

- `CompareResultsViewProvider` no longer owns item diff/path formatting rules.
- Compare Results item action helpers are testable without loading VS Code extension-host APIs.
- Existing Compare Results webview state, message dispatch, and rendering behavior remain unchanged.
- Focused Compare Results validation and full automated verification pass.

## Verification

- `npm run clean:test && npm run build && tsc -p ./tsconfig.test.json && node --test out-test/test/compareResultsItemActions.test.js out-test/test/compareResultsViewState.test.js out-test/test/compareResultsMessageHandler.test.js out-test/test/compareResultsWebview.test.js` passed with 15 tests.

## Risks

- Diff action behavior depends on preserving the same ref selection rules used by the provider-local implementation.
- Clipboard behavior depends on preserving rename-target path preference and selected item ordering.
