# 0.0.39 Compare Results Clipboard Boundary Hardening

## Goal

Continue the architectural bad-practices hardening work by moving Compare Results clipboard boundary behavior out of the editor panel provider.

## Problem

`CompareResultsViewProvider` still combined selected item lookup, copied text construction, and direct `vscode.env.clipboard` access for copy actions. That kept host-service access and selection policy inside panel lifecycle code.

## Scope

- Added `src/compareResults/clipboardActions.ts` as the owner for Compare Results copy-to-clipboard actions.
- Routed `CompareResultsViewProvider` through `copyCompareResultFileNames` and `copyCompareResultFullPaths`.
- Preserved existing behavior for:
  - ignoring empty or stale selections;
  - resolving selected items from the current Compare Results state;
  - writing file names and full paths in rendered view order;
  - lazily loading VS Code clipboard services only when no explicit services are supplied.
- Added focused tests for file-name copy, full-path copy, stale selections, and view-order preservation.

## Acceptance Criteria

- `CompareResultsViewProvider` no longer touches `vscode.env.clipboard` for Compare Results item copy actions.
- Clipboard behavior is testable without loading VS Code extension-host APIs.
- Existing Compare Results message dispatch, item diff actions, state mapping, and rendering behavior remain unchanged.
- Focused Compare Results validation and full automated verification pass.

## Verification

- `npm run clean:test && npm run build && tsc -p ./tsconfig.test.json && node --test out-test/test/compareResultsClipboardActions.test.js out-test/test/compareResultsItemActions.test.js out-test/test/compareResultsMessageHandler.test.js out-test/test/compareResultsWebview.test.js` passed with 14 tests.
- `npm test` passed with 381 tests. This includes `npm run build` through the test script.
- `git diff --check` passed after the Compare Results clipboard boundary extraction and verification artifact updates.

## Risks

- Copy actions still depend on the provider to pass the current state and selected IDs from the webview message.
- Manual Extension Development Host smoke testing should still confirm Compare Results copy file name and copy full path actions write the expected newline-delimited values.
