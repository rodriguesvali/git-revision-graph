# 1.0.2 Reload With Empty Cache

## Goal

Let users force a revision graph reload without reusing the persisted layout cache.

## Context

The revision graph persists projected layout positions to keep repeated loads stable and faster. When the cached layout becomes stale or visually suspicious, the user currently has no direct way to reload the graph from a clean layout cache without clearing extension storage manually.

The approved interaction follows the Chrome reload pattern: keep the normal `Reload Revision Graph` click behavior, and reveal an additional `With Empty Cache` option after holding the reload button for 500 ms.

## Scope

- Add a long-press menu to the existing graph reload toolbar button.
- Keep single-click reload unchanged.
- Add a webview message for `refresh-with-empty-cache`.
- Clear the in-memory projected layout cache and persisted workspace-state cache before running a full graph rebuild.
- Add focused tests for webview wiring, message validation/dispatch, message handling, and layout cache persistence.

## Non-Goals

- No command palette command, setting, or package contribution point.
- No reset of Git state, workspace changes, repository data, or secondary review views.
- No redesign of the graph toolbar or reload icon.

## Acceptance Criteria

- Clicking `Reload Revision Graph` still performs the normal reload.
- Holding `Reload Revision Graph` for 500 ms opens a menu with `With Empty Cache`.
- Choosing `With Empty Cache` clears layout cache state and then performs a full graph rebuild.
- The new message is accepted as an unscoped graph refresh action and remains valid while the graph is loading.
- Automated verification passes before release handoff.
