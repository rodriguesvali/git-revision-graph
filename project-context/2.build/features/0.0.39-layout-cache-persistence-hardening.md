# 0.0.39 Layout Cache Persistence Hardening

## Goal

Continue the architectural bad-practices hardening work by moving revision graph layout cache persistence out of the extension activation entrypoint.

## Problem

`src/extension.ts` mixed activation/bootstrap responsibilities with revision graph layout cache storage policy, including workspace state keys, debounce timers, restore normalization, persistence skipping, and error recovery. That made the activation entrypoint harder to reason about and left cache lifecycle behavior largely untested outside broad graph tests.

## Scope

- Added `src/revisionGraph/layout/cachePersistence.ts` as the owner for projected graph layout cache persistence.
- Moved workspace-state key ownership, restore normalization, debounce scheduling, persistence skipping, flush-on-dispose, and clear-on-error behavior into the new service.
- Kept `extension.ts` responsible only for creating the service, restoring it during activation, subscribing to cache change notifications, and disposing it with the extension lifecycle.
- Added focused unit tests for restore, normalized rewrite, unchanged-write skipping, serialization failure recovery, debounce, and dispose flush behavior.

## Acceptance Criteria

- Extension activation no longer owns projected graph layout cache persistence internals.
- Existing layout cache serialization and restore semantics are preserved.
- Debounced persistence and dispose-time flush remain part of the extension lifecycle.
- Focused graph/cache persistence validation and full automated verification pass.

## Verification

- `npm run clean:test && npm run build && tsc -p ./tsconfig.test.json && node --test out-test/test/revisionGraphLayoutCachePersistence.test.js out-test/test/revisionGraph.test.js` passed with 54 tests.

## Risks

- Dispose-time persistence is still asynchronous, matching the previous `void persist...` behavior.
- If workspace-state persistence fails repeatedly, the service keeps the previous clear-on-error behavior and logs through the injected warning service.
