# Refs-Only Merge Simplification

## Build Status

Build implementation complete for the `0.0.27` graph readability stabilization slice.

This artifact records the change that reduces visual distortion from local merge commits in the default refs-only graph. Version bump, package generation, and Marketplace publishing remain unapproved.

## Goal

Make the default revision graph easier to read after branches are merged into the local branch by avoiding merge-only connector cards while preserving the lines that show which visible branches were merged.

## User Workflow

- Users merge one or more branches into the local `master` or another current branch.
- Users open or refresh the revision graph in the default refs-only view.
- The graph should emphasize refs with fewer gray merge cards, while still drawing parent lines from visible merge commits to the visible branch tips they merged.
- Users can still enable "branching/merge commits" from the View menu to inspect the complete merge topology.

## Scope

- Stop promoting unreferenced merge commits as structural connector cards in refs-only projection.
- Preserve merge-parent edges from visible merge commits in refs-only graph projection.
- Apply the same refs-only simplification to `git --simplify-by-decoration` graph snapshots.
- Preserve full merge parents and hidden merge connector cards when `showBranchingsAndMerges` is enabled.

## Non-Goals

- Change Git merge workflows.
- Remove merge commits from Show Log.
- Hide ref-bearing commits from the graph.
- Add a new setting or persistent preference.
- Replace the current projection or ELK layout pipeline.

## Implementation Notes

- Structural connector promotion now treats merge commits as detailed-topology-only. Branching connectors can still be promoted in refs-only mode when needed for readable split points.
- Visible merge commits still project all parent edges, so merged branch lines remain connected even when an intermediate merge-only card is hidden.
- Git-simplified graph visibility now starts from displayable refs and opt-in branch/merge commits instead of keeping every simplified commit visible by default.
- Detailed mode keeps the previous complete merge topology behavior.
