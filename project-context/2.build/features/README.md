# Feature Build Artifacts

Use this directory for feature-scoped AAMAD notes.

Name files with a version or feature slug, for example:

- `0.0.18-tag-actions.md`
- `0.0.18-graph-filtering.md`
- `feature-full-history-windowing.md`

Each feature artifact should include:

- Goal
- User workflow
- Scope
- Non-goals
- Touched surfaces
- Acceptance criteria
- Test plan
- Manual validation
- Release-note impact
- Handoff notes

Keep these files concise. Link to broader docs such as `README.md` or `docs/revision-graph-parity-plan.md` instead of duplicating them.
