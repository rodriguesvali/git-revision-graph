# 2.0.0 Flow Governance Icon Badges

Status: Implemented; visual Extension Development Host validation complete
Updated: 2026-07-17

## Goal

Reduce the horizontal footprint of Flow Governance branch-kind badges and align their visual
language with VS Code without removing textual or assistive identification.

## UX Decision

- Replace `MAIN`, `REL`, `SYNC`, `PKG`, `FEAT`, `TASK`, `BUG`, `HOTFIX`, and `?` in graph cards
  with selected Codicon glyphs.
- Let the reference row retain the primary ref-state color signal; render graph badges as quiet
  tonal containers that inherit that row's foreground instead of adding saturated kind colors.
- Reuse the same tonal icon badge in Flow Governance reference tooltips; keep textual chips only
  for non-governed reference kinds such as remote, tag, and stash.
- Give each icon-only badge an accessible `<kind> branch type` label.
- Expose the complete kind name through the tooltip icon's accessible label.
- Keep unknown or unsupported kinds fail-soft with the question glyph and their supplied label.
- Use a high-contrast-only outline and no hover chrome because the badge is metadata, not a button.

## Scope And Constraints

- Keep rendering browser-only and CSP-safe through trusted inline SVG.
- Add no runtime dependency, resource root, font source, setting, command, protocol field, or Git
  behavior.
- Attribute the adapted Codicon glyph paths in `THIRD_PARTY_NOTICES.md`.
- Preserve Flow Governance-disabled rendering and every existing reference interaction.

## Acceptance Criteria

- Every classified graph reference renders the mapped icon without its old abbreviation.
- Graph badges are 18px tonal containers with 12px icons and no button-like border.
- Icons inherit the current reference-row foreground; Task receives an 11px optical adjustment.
- High-contrast themes receive a `--vscode-contrastBorder` outline without changing layout.
- Flow Governance tooltips reuse the same tonal icon badge; non-governed reference kinds retain
  their neutral textual chip.
- The badge exposes an accessible branch-type label and the nested SVG is decorative.
- The reference tooltip exposes `Main`, `Release`, `Feature`, and the other complete labels.
- All mappings and fallback behavior are covered by typed webview tests.

## Verification

- Focused typed webview tests passed 76/76.
- `npm run quality:check` passed with 223 production files and 2,110 functions.
- `npm run build`, all 751 tests, and `git diff --check` passed.
- `graphify update .` rebuilt 4,514 nodes, 8,865 edges, and 360 communities after the UX
  refinement.
- Visual Extension Development Host validation completed by maintainer confirmation on
  2026-07-17.

The first visual review exposed the graph's broad absolute-positioning rule for `svg` elements
also affecting badge glyphs. `.flow-badge-icon` now explicitly restores static positioning and an
automatic inset, keeping each glyph centered by its badge's inline-flex layout.

A second UX review found that saturated per-kind fills and dark borders made the icons resemble
miniature controls and competed with the reference-row colors. The graph badge treatment now
inherits the row foreground over a 12% tonal fill, has no normal-theme border, and uses a
high-contrast-only inset outline. Release uses `rocket` and Feature uses `lightbulb` for stronger
recognition at compact sizes. A follow-up visual review extended the same tonal icon treatment to
Flow Governance tooltips, replacing their remaining colored textual chip while retaining the full
kind in `aria-label`; non-governed reference kinds keep their neutral textual chip.
