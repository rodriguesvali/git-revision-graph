# Ref Actions Cohesion

## Goal

Start the `0.0.39` ref action cohesion work by splitting workflow families out of the broad `src/refActions.ts` module while preserving the existing public command-facing API.

## Scope

- Move reference compare workflows into `src/refActions/compare.ts`.
- Move checkout and branch creation workflows into `src/refActions/branches.ts`.
- Move branch publication and current branch sync/pull/push workflows into `src/refActions/currentBranch.ts`.
- Move local and remote tag workflows into `src/refActions/tags.ts`.
- Promote shared refresh preparation and remote-picking helpers into `src/refActions/shared.ts` for reuse across workflow families.
- Keep `src/refActions.ts` as the stable facade for compare, checkout, branch creation, current branch, and tag workflow exports.
- Preserve existing command, revision graph, and compare results imports.

## Acceptance Criteria

- `compareResolvedRefs` still uses repository `diffBetween` and opens Compare Results for non-empty changes.
- `compareResolvedRefWithWorktree` still uses repository `diffWith` and opens Compare Results for non-empty changes.
- Empty compare results still show the existing informational messages.
- Compare errors still surface the existing user-facing operation errors.
- `checkoutResolvedReference` still preserves current-branch short-circuit, clean-worktree/conflict guards, confirmation labels, and direct checkout behavior.
- `createBranchFromResolvedReference` still preserves branch-name suggestions, validation, remote tracking, existing-branch overwrite handling, and cancel behavior.
- `publishLocalBranchResolvedReference` still preserves remote selection, inherited-upstream messaging, conflict guards, refresh timing, and authentication fallback behavior.
- Current branch sync, pull, and push still preserve upstream guards, clean-worktree/conflict guards, force-push confirmation, refresh timing, and Source Control reveal behavior.
- `createTagFromResolvedReference` still validates duplicate and invalid tag names before creating tags.
- `pushTagResolvedReference` and `deleteRemoteTagResolvedReference` still preserve remote selection, conflict guards, confirmations, and authentication fallback behavior.
- Existing imports from `src/refActions.ts` keep working.
- `npm run build` passes.
- `npm test` passes.

## Risks

- Moving compare, checkout, branch creation, current branch, or tag workflows can drift the public `refActions.ts` facade if exports are not preserved.
- Compare Results behavior can drift if labels, empty-state messages, or error text change during extraction.
- Checkout and branch creation can drift if conflict guards, branch suggestions, tracking behavior, existing-branch handling, or user-facing messages change during extraction.
- Current branch workflows can drift if upstream checks, force-push handling, authentication messages, Source Control reveal, or refresh timing change during extraction.
- Tag actions can drift if conflict guards, remote selection, confirmations, authentication messages, or refresh timing change during extraction.
