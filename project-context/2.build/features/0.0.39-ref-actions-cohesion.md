# Ref Actions Cohesion

## Goal

Start the `0.0.39` ref action cohesion work by splitting workflow families out of the broad `src/refActions.ts` module while preserving the existing public command-facing API.

## Scope

- Move reference compare workflows into `src/refActions/compare.ts`.
- Move local and remote tag workflows into `src/refActions/tags.ts`.
- Promote shared refresh preparation and remote-picking helpers into `src/refActions/shared.ts` for reuse across workflow families.
- Keep `src/refActions.ts` as the stable facade for compare and tag workflow exports.
- Preserve existing command, revision graph, and compare results imports.

## Acceptance Criteria

- `compareResolvedRefs` still uses repository `diffBetween` and opens Compare Results for non-empty changes.
- `compareResolvedRefWithWorktree` still uses repository `diffWith` and opens Compare Results for non-empty changes.
- Empty compare results still show the existing informational messages.
- Compare errors still surface the existing user-facing operation errors.
- `createTagFromResolvedReference` still validates duplicate and invalid tag names before creating tags.
- `pushTagResolvedReference` and `deleteRemoteTagResolvedReference` still preserve remote selection, conflict guards, confirmations, and authentication fallback behavior.
- Existing imports from `src/refActions.ts` keep working.
- `npm run build` passes.
- `npm test` passes.

## Verification

- Focused ref action/ref command/revision graph workflow tests passed with 98 tests after extracting compare workflows.
- `npm test` passed with 312 tests after extracting compare workflows.

- `npm run build` passed after extracting tag workflows.
- `npm test` passed with 312 tests after extracting tag workflows.
- `git diff --check` passed after extracting tag workflows.

## Risks

- Moving compare or tag workflows can drift the public `refActions.ts` facade if exports are not preserved.
- Compare Results behavior can drift if labels, empty-state messages, or error text change during extraction.
- Tag actions can drift if conflict guards, remote selection, confirmations, authentication messages, or refresh timing change during extraction.
