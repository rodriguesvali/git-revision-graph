# Webview Virtualized Graph Rendering

## Goal

Reduce DOM work for large revision graphs by rendering only the graph nodes and edges that intersect the current webview viewport plus overscan, while keeping the full graph model available for layout, minimap, centering, selection, and persisted node offsets.

## Scope

- Keep d3-dag layout calculation unchanged.
- Keep rendering inside the existing revision graph webview; no new extension runtime or Python dependency.
- Virtualize DOM creation for graph cards and edge paths after the ready scene is loaded.
- Re-render the visible window on scroll, resize, zoom, and metadata patches.
- Keep the minimap based on the full graph layout rather than the visible DOM subset.

## Implementation Notes

- Added `renderVirtualScene` and `scheduleVirtualSceneRender` in the webview bootstrap script.
- The visible window is computed from current scroll position, zoom, scene placement offsets, and an overscan margin.
- Graph caches are refreshed after each virtual DOM swap so delegated handlers, selection state, search highlights, and edge updates continue to target mounted elements only.
- Layout helpers now use the full `graphNodes` model for graph bounds, HEAD centering, persisted offsets, and minimap rendering.
- Metadata patches now refresh the virtual window instead of replacing individual mounted nodes, because an affected node may be outside the current DOM window.

## Acceptance Criteria

- Initial graph render mounts only the current virtual window.
- Scroll and resize update the mounted graph window.
- Zoom preserves viewport behavior and forces a virtual window refresh.
- Minimap still represents the complete graph.
- Metadata patches still preserve viewport and selection when requested.
- Existing graph actions remain delegated through the webview layers.

## Residual Risk

- Manual Extension Development Host validation is still recommended on a large repository to observe scroll smoothness, search/selection behavior across offscreen nodes, and minimap navigation with real VS Code webview rendering.
