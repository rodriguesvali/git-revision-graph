# 0.0.37 Remote Checkout Current Branch Overwrite

## Goal

Complete the remote branch checkout overwrite workflow by adding an explicit `Override branch if exists` option, similar to TortoiseGit, for existing local branch names.

## Scope

- Detect when the local branch name selected during remote checkout matches the current branch.
- Offer a native VS Code QuickPick with editable branch name and `Override branch if exists` unchecked by default in the same first checkout prompt.
- Reset an existing local branch to the selected remote reference only when `Override branch if exists` is selected.
- Keep the current branch checked out and configure it to track the selected remote branch.
- Preserve existing clean-workspace and conflict guards.

## Non-Goals

- Automatically overwriting a local branch when the override option is not selected.
- Changing checkout behavior for local branches, tags, or commits.
- Adding a custom webview dialog for checkout options.

## Acceptance Criteria

- Checking out `origin/main` shows `main` in an editable field and an unchecked `Override branch if exists` option in the same prompt.
- Leaving the option unchecked does not reset the local branch.
- Selecting the option resets current `main` to `origin/main` and configures upstream tracking without an intermediate checkout.
- Canceling the options picker performs no checkout, reset, or upstream change.
- Dirty workspaces and unresolved conflicts remain blocked.
- `npm run build`, focused tests, and `npm test` pass.
