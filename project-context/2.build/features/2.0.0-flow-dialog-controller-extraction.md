# 2.0.0 Flow Dialog Controller Extraction

Status: Implemented; automated validation complete
Updated: 2026-07-17

## Goal

Reduce the responsibility and change surface of the revision graph interaction script by moving
the Flow Governance branch, Pull Request context, and equalization dialogs into typed runtime
controllers without changing their user-visible behavior or host protocol.

## Scope And Design

- Keep `interactions.ts` responsible for interaction orchestration and adaptation of current graph
  state to dialog inputs.
- Give each dialog one typed controller that owns its DOM lifecycle, local state, validation,
  focus behavior, and event handlers.
- Inject context-menu closing and host-message callbacks into the controllers so they do not
  depend directly on the global VS Code API.
- Keep branch validation, Pull Request eligibility warnings, and equalization-origin selection in
  pure helpers with direct runtime tests.
- Preserve the existing `show-flow-branch-form` and `show-flow-pr-context` host-message paths and
  all existing outbound message factories.

## Non-Goals

- No command, menu, contribution point, setting, protocol, Git behavior, or dependency changes.
- No visual redesign of the dialogs.
- No package, version, VSIX, or Marketplace operation.

## Acceptance Criteria

- `interactions.ts` no longer creates or stores DOM state for the three Flow Governance dialogs.
- Dialog controllers depend on injected callbacks rather than the global host API.
- Runtime contracts type Pull Request targets and dialog inputs without custom properties on DOM
  elements.
- Existing Flow Governance webview behavior remains covered, and pure dialog rules have focused
  tests.
- The code-quality baseline for `interactions.ts` is ratcheted down from 1,756 to 1,040 lines.
- Isolated webview type checks, the production bundle, the complete test suite, quality checks,
  whitespace checks, and the knowledge graph all pass.

## Release And Rollback Notes

The change is internal and does not alter Marketplace metadata. If a dialog regression is found,
the three controllers and their adapters can be reverted together without reverting Flow
Governance host workflows or protocol definitions.
