# 1.5.9 Reference Tooltips

Status: Published
Last updated: 2026-07-09
Baseline: published `1.5.8`; cosmetic source reference `6f3342a90618586b429bc41114231c3bfc0fd9c6`

## Goal

Bring the improved reference-tooltip presentation from the 2.0.0 development line into `1.5.9`
without introducing Flow Governance behavior or branch-kind classification.

## User Workflow

1. Open the revision graph and hover or keyboard-focus an individual reference.
2. Review its neutral reference kind, full name, optional local Git branch description, commit
   subject, short hash, author, date, and change statistics in a compact themed detail popover.
3. Move the pointer into the popover to copy the commit hash or open the commit on GitHub.
4. Use `Enter` or `Space` while focused to select the reference.

## Scope

- Add the compact reference tooltip, viewport-aware placement, keyboard focus, and themed styling.
- Keep the detail popover open while the pointer moves into it and expose hash-copy and GitHub
  actions through validated host messages.
- Load commit file/insertion/deletion statistics on demand and cache them by immutable commit hash
  for the active repository.
- Read optional local `branch.<name>.description` values without failing graph loading when Git
  config has no descriptions or cannot be read.
- Attach descriptions only to local branch and current HEAD references.
- Update version, README, changelog, tests, and release readiness for `1.5.9`.

## Non-goals

- No Flow Governance branch-kind badge, classification, configuration, commands, or policy.
- No new commands, settings, views, dependencies, or Marketplace contribution points.
- No packaging, publication, or Marketplace deployment in this build step.

## Touched Surfaces

- Revision graph state/reference serialization and local Git metadata loading.
- Revision graph webview markup, interactions, accessibility, and styles.
- Shared webview display helpers for tooltip dates and copy-hash icon actions.
- Show Log reference badges in commit rows and commit tooltips.
- Unit tests and `1.5.9` product/release documentation.

## Acceptance Criteria

- Hovering or focusing a ref shows only that reference's details and avoids the competing native
  multi-reference node tooltip.
- The tooltip includes the neutral ref kind, full name, optional description, commit subject,
  short hash, author, date, and available change statistics.
- The user can move the pointer into the detail popover, copy the full commit hash, and request the
  GitHub commit URL; repositories without a supported GitHub remote receive existing native
  informational feedback.
- Change statistics are fetched lazily, cached, bounded, and rejected for unknown graph commits.
- No Flow Governance branch-kind information or CSS class is rendered.
- Missing or unreadable branch descriptions do not block graph loading.
- `npm run build`, `npm test`, and `git diff --check` pass.

## Risks

- A lightweight local Git config read occurs during graph state construction.
- The first tooltip visit for a commit performs one bounded local `git show --shortstat`; later
  visits use the webview cache.
- Tooltip placement and theme contrast still require Extension Development Host smoke validation.

## Test Plan

- Unit-test branch-description and short-stat parsing plus state attachment boundaries.
- Assert tooltip shell, interactive dismissal, content, actions, placement, accessibility hooks,
  message authorization, host dispatch, and governance exclusion.
- Run the full build and test suites plus whitespace validation.
- Keep shared tooltip date/icon behavior covered by focused helper tests rather than duplicating
  implementation-specific assertions across each webview shell.

## Manual Validation

- In an Extension Development Host, hover and keyboard-focus local, remote, tag, and stash refs;
  move the pointer into the popover and exercise both actions.
- Verify long names, multiline descriptions, viewport edges, scrolling, resizing, dark/light themes,
  stats loading/cache behavior, ref selection, multi-ref nodes, repositories without branch
  descriptions, and repositories with and without GitHub remotes.
