# Reset Current Workspace

## Build Status

Build implementation complete for the `0.0.29` current-branch workspace reset feature.

Version bump, package generation, and Marketplace publishing remain unapproved.

## Goal

Let users reset the workspace from the context menu of the current local branch in the revision graph.

## User Workflow

A user opens the graph context menu on the current local branch and chooses one of two destructive reset actions:

- `Reset Workspace to HEAD`
- `Reset Workspace and Remove Untracked Files`

The extension asks for explicit confirmation before changing the workspace.

## Scope

- Show reset actions only for the current local `HEAD` reference when the workspace is dirty and not in a conflicted merge.
- Add a host message for reset requests with an explicit `includeUntracked` flag.
- Validate reset requests against the current ready graph state.
- Run `git reset --hard HEAD` for tracked and staged changes.
- Run `git clean -fd` only when the user chooses the untracked-removal action.
- Refuse reset while a conflicted merge is active; users should abort or resolve the merge first.
- Refresh workspace state after reset through an overlay patch.

## Non-Goals

- Resetting non-current branches.
- Resetting the branch ref to another revision.
- Removing ignored files.
- Bypassing confirmation.
- Running reset during conflicted merges.

## Acceptance Criteria

- Current local branch context menu includes both reset actions in the destructive section only when the workspace is dirty and not in a conflicted merge.
- Reset without untracked removal discards tracked and staged changes but keeps untracked files.
- Reset with untracked removal also removes untracked files.
- Both actions require modal confirmation.
- Reset requests are accepted only for a ready graph state with a known current `HEAD` reference.
- Conflicted merges block reset and preserve the conflict recovery flow.

## Manual Validation

- In a clean repository, create tracked/staged changes, use `Reset Workspace to HEAD`, and confirm tracked changes are discarded.
- Create an untracked file and confirm `Reset Workspace to HEAD` keeps it.
- Use `Reset Workspace and Remove Untracked Files` and confirm the untracked file is removed.
- Trigger a conflicted merge and confirm reset is refused until merge is aborted or resolved.
