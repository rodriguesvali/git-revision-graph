# 0.0.39 Git Error Detail Helper Code Quality

## Goal

Keep Git-like error inspection in one shared module instead of spreading small property checks across action helpers.

## Problem

`refActions/shared.ts` inspected `stderr` directly to detect missing upstream configuration, while `src/errorDetail.ts` already owned Git-like error normalization and operation error formatting. This split made error handling rules harder to discover and easier to drift.

## Scope

- Added `isMissingUpstreamConfigurationError` to `src/errorDetail.ts`.
- Re-exported the helper from `src/refActions/shared.ts` to preserve existing internal import paths.
- Added direct coverage for error detail formatting, non-interactive authentication detection, and missing upstream detection in `test/errorDetail.test.ts`.

## Acceptance Criteria

- Missing-upstream detection lives with the other Git-like error helpers.
- Existing ref action callers and tests continue to import the helper through `refActions/shared`.
- Focused error/ref action validation and full automated verification pass.

## Verification

- `npm run build` passed after moving missing-upstream detection.
- `npm run clean:test && npm run build && tsc -p ./tsconfig.test.json && node --test out-test/test/errorDetail.test.js out-test/test/refActions.test.js` passed with 87 tests.
- `npm test` passed with 392 tests. This includes `npm run build` through the test script.
- `git diff --check` passed after the Git error detail helper cleanup and verification artifact updates.

## Risks
