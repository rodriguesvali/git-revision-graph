# 1.0.1 Force Push VS Code Git API Hotfix

## Goal

Fix current-branch force push and force-with-lease push so the extension uses the built-in VS Code Git API force parameter instead of direct non-interactive Git CLI execution.

## Context

The current-branch `Push` action offers normal, force-with-lease, and force modes. Normal push uses `repository.push(...)` from the built-in VS Code Git API, but force modes currently call `execGitWithResult(...)` with `git push --force-with-lease` or `git push --force`.

A reported failure shows the direct CLI path cannot prompt for HTTPS credentials:

```text
fatal: could not read Username for 'https://github.com': terminal prompts disabled
```

Context7 and the official VS Code Git API source confirm that the supported `Repository.push` signature includes:

```ts
push(remoteName?: string, branchName?: string, setUpstream?: boolean, force?: ForcePushMode): Promise<void>;
```

The official `ForcePushMode` enum includes `Force`, `ForceWithLease`, and `ForceWithLeaseIfIncludes`. This support exists in the VS Code `1.90.0` API baseline targeted by the extension.

## Scope

- Update the local `src/git.ts` contract with `ForcePushMode` and the fourth `Repository.push` parameter.
- Replace direct current-branch force-push CLI execution with `repository.push(remoteName, branchName, false, ForcePushMode.*)`.
- Preserve the existing push-mode selection, destructive confirmation, upstream branch normalization, refresh preparation/cancellation behavior, and user-facing messages.
- Add or update focused tests to prove the selected force mode reaches the Git API adapter.

## Non-Goals

- No broad push workflow redesign.
- No new commands, contribution points, settings, dependencies, or Marketplace metadata changes.
- No changes to tag push, remote delete, branch publish, pull, sync, fetch, checkout, merge, or reset workflows unless required by the hotfix.
- No VSIX packaging or Marketplace publication in the implementation step without explicit maintainer approval.

## Acceptance Criteria

- Force-with-lease maps to the VS Code Git API `ForcePushMode.ForceWithLease`.
- Force maps to the VS Code Git API `ForcePushMode.Force`.
- Current-branch force modes no longer call direct `git push --force...` through `execGitWithResult`.
- Normal current-branch push behavior remains unchanged.
- Regression tests cover the force-mode API dispatch.
- `npm run build`, `npm test`, and `git diff --check` pass before release packaging.

## Risks

- Force-push actions are destructive; the existing confirmation must remain intact.
- Authentication behavior can vary by remote provider and credential helper, so manual validation should include an HTTPS remote when practical.
- The local API type update should stay minimal and avoid pulling in unrelated `vscode.git` API surface.
