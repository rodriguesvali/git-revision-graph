# Context menu item standardization

## Goal

Make context-menu item highlighting consistent across the revision graph, Show Log, and Compare Results webviews using the revision graph menu design as the visual baseline.

## Scope

- Promote `context-menu-item` as the shared visual class for context-menu entries.
- Remove the legacy `context-menu-button` hook and use `context-menu-item` for both menu item styling and initial focus.
- Preserve `context-menu-parent` only for submenu-parent behavior.
- Use the graph-style rounded filled hover/focus treatment rather than the thin leading-border treatment.
- Keep webview-specific theme variables for hover color strength while aligning the visible interaction pattern.

## Acceptance criteria

- Revision graph menu entries render with `context-menu-item`.
- Show Log and Compare Results menu buttons and submenu parents include `context-menu-item`.
- No production source uses `context-menu-button`.
- Menu hover/focus states use a subtle filled rounded highlight, matching the preferred graph design.
- Existing context-menu keyboard focus and action dispatch behavior remain unchanged.
