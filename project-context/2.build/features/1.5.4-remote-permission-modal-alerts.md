# 1.5.4 Remote Permission Modal Alerts

## Status

Implemented.

## Objective

Restore blocking error presentation for remote Git authorization failures that were being surfaced as standard VS Code notifications.

## Scope

- Detect remote permission and protected-branch failures from Git error text and metadata.
- Keep ordinary sync and pull failures nonblocking so the revision graph action state still clears quickly.
- Show modal errors for server-side authorization failures in remote-mutating workflows:
  - current-branch sync
  - current-branch push
  - branch publish
  - tag push
  - remote tag delete
  - remote branch delete

## Implementation Notes

- Added `isRemotePermissionDeniedError(...)` in `src/errorDetail.ts`.
- `syncCurrentHeadWithUpstream` and `pullCurrentBranchFromUpstream` only await the error message when the failure is classified as a remote permission problem.
- Remote mutation actions pass `{ modal: true }` for permission/protected-branch failures while preserving existing toaster behavior for generic Git failures.
