# 1.0.0 Remote Checkout Overwrite Confirmation

## Goal

Require an explicit destructive confirmation before a remote checkout with `Override branch if exists` resets an existing local branch.

## Context

The remote checkout prompt lets users select `Override branch if exists` when checking out refs such as `origin/main`. That option currently proceeds directly to resetting the matching local branch, including the current branch, without a second modal warning.

## Scope

- Preserve the existing remote checkout options prompt and default unchecked override option.
- When override is selected and the local branch exists, show a modal confirmation before any branch reset.
- Use a distinct confirmation label for current-branch overwrite.
- Canceling the confirmation performs no reset, checkout, upstream change, or branch creation.

## Non-Goals

- No change to local branch checkout, tag checkout, or commit checkout behavior.
- No change to the Git implementation used for branch reset.
- No new webview dialog.

## Acceptance Criteria

- Existing non-current local branch overwrite requires confirmation.
- Current local branch overwrite requires confirmation.
- Canceling either confirmation leaves repository mutation calls untouched.
- Existing create/checkout-without-overwrite behavior remains unchanged.
