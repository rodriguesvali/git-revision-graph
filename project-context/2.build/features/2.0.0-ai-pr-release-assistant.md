# 2.0.0 AI PR and Release Assistant

Status: Implemented (manual Extension Development Host smoke pending)

## Goal

Let users explicitly improve text in the existing Flow Governance Promotion Pull Request Context and
Start New Release forms without replacing their deterministic defaults or changing Git safety rules.

## User workflow

1. Open a Promotion Pull Request Context form. The deterministic title and description appear as today.
2. Optionally select the AI icon to the right of either field. The improved field replaces only that
   visible value; copy and provider handoff use the improved context.
3. Open Start New Release, enter the normal release name and description, and optionally improve the
   description through the AI icon before submitting.
4. Continue editing or retrying under explicit user control. No model request runs on form open.
5. In Promotion PR Context, manually edit either generated field at any time. Description improvement
   uses bounded documentation changes between the selected target and source refs when available.

## Scope

- Add icon-only, tooltip-labelled AI actions beside PR title, PR description, and release description.
- Keep the existing deterministic PR context and user-entered release form as the initial source.
- Use an optional GitHub Copilot model through the VS Code Language Model API only after a button click.
- Bound all inbound form text and model output to the existing field limits.
- Preserve branch/ref facts, improve clarity and structure, and prevent the model from inventing tests,
  tickets, behavior, or release claims.
- Treat form text and ref labels as untrusted prompt data.
- Cancel superseded or closed-form requests and reject stale responses in both host and browser.
- Keep accepted PR text in host-owned transient state and validate the bounded visible values sent by
  explicit copy, AI, and provider-handoff actions.
- Allow bounded manual edits to PR title/description and pass the exact visible values through the
  validated copy, AI, and provider-handoff intents.
- Enrich PR descriptions only with a bounded unified diff of allowlisted project documentation:
  `README.md`, `CHANGELOG.md`, and approved `project-context` define/build/deliver paths.
- Fit prompts against each available model's tokenizer and input limit. Prefer an integral prompt in
  provider order; otherwise use the largest available input window and progressively reduce only the
  optional documentation context.

## Non-goals

- No automatic generation on dialog open, automatic acceptance, background rewriting, persistent text,
  telemetry, provider API, PR creation, release publication, branch mutation, or Git operation.
- No AI assistance for feature, task, bug, hotfix, or equalization forms in this slice.
- No dependency, command, setting, contribution point, minimum VS Code version, or package-version change.

## Acceptance criteria

- Existing forms populate and submit normally when AI is unused.
- PR title and description remain editable after deterministic generation or AI improvement.
- PR title/description and release description expose accessible icon actions on the right.
- Loading disables only the affected field action and preserves form usability.
- A successful response replaces only its exact field and current form request.
- Closing, changing target, opening another form, switching repository, or retrying prevents stale text
  from being applied.
- Model unavailable, denied, blocked, or empty-result cases restore the action and present concise
  feedback without closing the form.
- PR copy actions and supported hosted-provider URLs use improved title/body when present.
- PR description prompts request a delivery summary, concrete documented changes, and documented
  verification while stating uncertainty instead of inventing missing facts.
- Oversized documentation context is reduced automatically without dropping the user's form values or
  failing the request when the essential prompt still fits an available Copilot model.
- Protocol validation, authorization, host-message validation, prompts, orchestration, browser
  controllers, and provider URL overrides have deterministic tests.

## Architecture

- `src/revisionGraph/flow/aiTextAssistant.ts`: pure contracts, prompt policy, bounds, and output shaping.
- `src/workbenchAiFlowTextAssistant.ts`: optional VS Code Language Model adapter.
- `src/revisionGraph/flow/aiTextWorkflow.ts`: host request lifecycle, transient PR context, cancellation,
  stale guards, and result messages.
- `src/revisionGraph/flow/aiTextDocumentContext.ts`: allowlisted, bounded, cancellable documentation
  diff loading for PR descriptions only.
- `src/revisionGraph/flow/aiTextPromptFitting.ts`: model-aware full-prompt selection and optional
  documentation reduction using each model's tokenizer and input ceiling.
- Existing Flow Governance workflow and message boundaries own composition and authorization.
- `src/revisionGraph/webview/runtime/flowAiTextUi.ts`: shared icon action and busy-state presentation.
- Existing PR and release form controllers own field values, request IDs, close/reset behavior, and
  application of matching host results.

## External contract evidence

- Context7 library: `/websites/code_visualstudio_api`.
- `selectChatModels` may return multiple or no models; `maxInputTokens` is the maximum request input,
  and `countTokens` uses model-specific tokenization and accepts cancellation.
- Current documentation confirms `sendRequest` may require consent and must be invoked only from an
  explicit user action, accepts a `CancellationToken`, and can fail for `NoPermissions`, `NotFound`, or
  `Blocked`.
- Exact response streaming compatibility remains governed by pinned `@types/vscode` `1.90.0`, where
  `LanguageModelChatResponse.text` is the supported async text stream.

## Risks and mitigations

- Hallucinated facts: prompt preserves supplied facts, forbids invention, and improves only existing text.
- Prompt injection: labels, field text, and documentation diffs are delimited and explicitly treated as
  untrusted data rather than instructions.
- Stale forms/repositories: request identity, surface/field matching, cancellation, and repository checks.
- Provider handoff mismatch: transient host PR context supplies both copy and URL generation.
- Excessive text: existing field limits plus prompt/output caps.
- Model-window mismatch: try the complete prompt across available models, then reduce only optional
  documentation against the largest windows; fail only when the essential prompt cannot fit.
- Accidental source disclosure: only explicit documentation pathspecs are loaded; source code and
  arbitrary repository files are excluded before Git reads.
