# Reference Deletion Refresh

## Build Status

Build implementation complete for the `0.0.30` reference deletion refresh correction.

Version bump, package generation, and Marketplace publishing remain unapproved.

## Goal

Restore visual correctness after deleting local branches from the revision graph by using a full graph rebuild instead of an optimistic reference-removal overlay.

## User Workflow

A user deletes a local branch from the graph context menu. After confirmation and successful Git deletion, the graph should reload from repository state so cards, refs, layout, selection, and viewport behavior match the current repository.

## Scope

- Replace the local branch deletion direct reference patch with a `full-rebuild` refresh request.
- Apply the same rebuild behavior for normal delete and force delete.
- Keep existing branch deletion confirmations, tracked-remote explanation, force-delete fallback, and error handling.
- Keep workspace-state `overlay-patch` behavior for merge/reset status updates.
- Remove the now-unused direct reference-patch refresh contract from controller/state/types/tests.

## Non-Goals

- Changing tag or remote branch deletion behavior.
- Changing Git mutation APIs or confirmation text.
- Removing metadata patch behavior for checkout/sync/repository events.
- Publishing or bumping the extension version.

- Successful forced local branch deletion schedules a full graph rebuild.
- Direct reference patch types/helpers are removed when no active workflow uses them.
- Existing workspace-state overlay patch workflows remain unchanged.
- Automated build and test verification pass.
