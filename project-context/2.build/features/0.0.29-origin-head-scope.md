# Origin HEAD Graph Scope

## Build Status

Build implementation complete for the `0.0.29` candidate scope.

Version bump, package generation, and Marketplace publishing remain unapproved.

## Goal

Add an `origin/HEAD` graph scope that behaves like `Current Branch` with descendants enabled, but uses the repository's default remote trunk as the graph root.

## User Workflow

A user opens a repository where the current working branch may be a feature branch, selects `origin/HEAD` in the graph scope selector, and sees the ancestry of the default remote branch, typically `origin/main` or `origin/master`, plus visible descendant refs that grow from that trunk, without switching branches.

## Viability

Feasible with low-to-moderate implementation risk.

The current graph architecture already separates:

- Git log source arguments in `src/revisionGraph/source/graphGit.ts`.
- Projection scope filtering in `src/revisionGraph/projection/graphProjection.ts`.
- Projection option normalization in `src/revisionGraphTypes.ts`.
- Webview scope selection in `src/revisionGraphWebview.ts` and `src/revisionGraph/webview/script/bootstrap.ts`.

The existing `Current Branch` scope is a close template. The main difference is root resolution: `Current Branch` can use `HEAD`, while this feature must resolve the symbolic remote default branch.

## Proposed Scope

- Add a new graph scope option labeled `origin/HEAD`.
- Extend `RevisionGraphProjectionOptions.refScope` with the new `remoteHead` value.
- For Git log loading, load all refs so descendant refs can be projected.
- For projection, collect ancestors from commits decorated with `origin/HEAD`, with fallback anchors for `origin/main` and `origin/master`, plus visible ref tips that descend from that trunk.
- Preserve existing tag, remote branch, and stash visibility toggles.
- Make descendant refs core behavior for both `Current Branch` and `origin/HEAD`, without a separate `View` menu toggle.
- Fall back gracefully when `origin/HEAD` is missing or unresolved.

## Non-Goals

- Changing the default graph scope.
- Fetching or modifying remote default-branch configuration automatically.
- Supporting arbitrary remote selection in the first slice.
- Full-history rendering beyond the existing bounded window.
- Adding a separate `Show origin/HEAD Descendants` toggle; descendants are part of this scope by default.
- Changing compare, checkout, merge, delete, sync, or Show Log workflows.

## Technical Notes

- Local validation in this repository shows `git symbolic-ref refs/remotes/origin/HEAD` resolves to `refs/remotes/origin/main`.
- `git log origin/HEAD --topo-order --simplify-by-decoration --decorate=short` works and decorates the tip with `origin/HEAD`.
- `buildRevisionGraphRefKinds` already treats `origin/HEAD` as a remote ref when present in Git API refs or decorations.
- Projection reuses the descendant-ref flow from `Current Branch`, but applies it unconditionally for `origin/HEAD`.
- Missing `origin/HEAD` is handled by projection fallback anchors for `origin/main` and `origin/master`.
- When the `origin/HEAD` scope is active, the default remote-trunk anchor remains available to projection even if remote branch labels are hidden.

## Risks

- Some repositories do not have a local `refs/remotes/origin/HEAD`.
- Some repositories do not use a remote named `origin`.
- The VS Code Git API may not always expose symbolic remote refs consistently, so Git log decorations and targeted Git CLI resolution may need to be the source of truth.
- If `showRemoteBranches` is disabled, the `origin/HEAD` root could be filtered out visually unless the projection keeps the root commit visible while hiding the remote label.

## Acceptance Criteria

- The scope selector includes `origin/HEAD`.
- Selecting `origin/HEAD` loads and projects the ancestry of the default remote trunk without changing the current branch.
- Descendant refs from `origin/HEAD` are shown by default when they satisfy the current visibility toggles.
- Repositories with `origin/HEAD -> origin/main` show the same ancestry as `origin/main`.
- Repositories with `origin/HEAD -> origin/master` show the same ancestry as `origin/master`.
- The former `Show Current Branch Descendants` view option is no longer rendered.
- Missing or unresolved `origin/HEAD` does not crash graph loading.
- Automated tests cover Git log args, projection behavior, option validation, option normalization, and webview scope wiring.
