# 1.5.2 Windows Test Runner Hotfix

## Status

Implementation complete and verified locally with Node 20 and the workspace Node runtime.
Package version remains `1.5.1` pending explicit release approval; `windows-latest` remains
the final platform proof.

## Problem

The Node 20 `windows-latest` GitHub Actions job fails after TypeScript test compilation:

```text
Could not find '...\\out-test\\test\\*.test.js'
```

The `npm test` script passes `out-test/test/*.test.js` to `node --test`. POSIX shells
expand that glob before starting Node, but Windows `cmd.exe` passes it literally. Node 20
then looks for a file named `*.test.js` and exits with code 1.

## Goal

Run the same compiled `*.test.js` files on Linux, Windows, and macOS without relying on
shell-specific glob expansion.

## Scope

- Add a dependency-free Node runner that enumerates compiled `*.test.js` files.
- Pass the sorted explicit file list to the Node test runner.
- Keep helper files such as `fakes.js` and `gitFixture.js` out of the test entrypoint list.
- Update the `npm test` script to invoke the cross-platform runner.
- Add manifest regression coverage that rejects the shell glob.
- Keep `test:platform` unchanged because it already uses explicit file paths.

## Acceptance Criteria

- `npm test` contains no shell-expanded test-file glob.
- The runner fails clearly when no compiled test entrypoints exist.
- Only files ending in `.test.js` are passed to `node --test`.
- The full suite retains all existing tests and adds the manifest regression.
- Linux local verification passes; Windows GitHub Actions remains the final platform proof.

## Verification Plan

- `npm run build`
- `npm test`
- `npm run test:platform`
- `git diff --check`
- Re-run the GitHub Actions Node 20 matrix and confirm `windows-latest` passes.

## Tooling Constraint

The GitHub CLI is not installed in the current environment, so the failing Actions run
cannot be re-fetched here. Root-cause evidence comes from the maintainer-provided log and
the checked-in `.github/workflows/verify.yml` matrix.

## Implementation

- Added `scripts/run-compiled-tests.mjs` to enumerate sorted compiled `.test.js` files and
  invoke `node --test` with explicit paths.
- Updated `npm test` to invoke the runner instead of passing a shell glob.
- Added package-manifest coverage for the exact cross-platform test command and runner file.
- Kept `test:platform` unchanged because its explicit file list is already shell-independent.
