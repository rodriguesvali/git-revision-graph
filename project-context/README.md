# Project Context

Status: Active
Last consolidated: 2026-06-30

This directory holds AAMAD planning, build, and release context for the published `Git Revision Graph` VS Code extension.

## Read First

- `1.define/prd.md`: current product requirements and product boundaries.
- `1.define/sad.md`: current architecture and technical constraints.
- `1.define/open-questions.md`: unresolved decisions only.
- `2.build/features/README.md`: active feature artifact policy.
- `3.deliver/release.md`: current release readiness state.
- `docs/product-evolution-roadmap.md`: next product evolution themes.

## Historical Context

Historical feature notes, release notes, prioritization plans, and superseded define artifacts live under `archive/`. They remain available for traceability, but they are not current requirements unless a maintainer explicitly reopens them.

## Maintenance Rule

After completing feature or release work, promote durable conclusions into the active PRD/SAD and archive the operational artifact. Avoid letting active context become a chronological log.
