# 0.0.37 Compare Results Review UX

## Goal

Turn Compare Results from a sparse file-card list into a compact review queue that makes comparison context, change types, and primary actions immediately understandable.

## Scope

- Show the comparison source and target as an explicit directional header.
- Show clear changed-file, visible-file, and selected-file counts.
- Add status summary filters for modified, added, deleted, renamed, and other changed files.
- Replace repeated cards with a dense status/file/folder/actions list.
- Keep `Open Diff` in double-click and context-menu flows, with a centered `...` row action for additional commands.
- Keep worktree restore, compare-with-worktree, clipboard, keyboard selection, and multi-selection behavior.
- Select a single result automatically without opening its diff.
- Show original and target paths for renamed files.
- Keep the layout usable at narrower editor widths.

## Non-Goals

- Adding reviewed/unreviewed persistence.
- Rendering inline diff contents inside Compare Results.
- Changing Git diff or restore behavior.
- Changing the main Revision Graph or Show Log.

## Acceptance Criteria

- Users can identify the comparison source, target, and total changed-file count without reading each row.
- Users can filter results by change status and file text.
- Result rows are compact and expose status, file, folder, and a centered `...` actions menu.
- Renamed files show both original and target paths.
- Single-file results are selected automatically.
- Existing diff, worktree comparison, restore, clipboard, keyboard, and multi-selection workflows remain available.
- Build and tests pass.
