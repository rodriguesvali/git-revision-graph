# Show Log State Lookup Cohesion

## Goal

Continue the `0.0.39` architectural cohesion work by separating Show Log state lookup rules from the editor panel provider actions.

## Scope

- Add `src/showLog/stateLookup.ts` as the owner of visible repository lookup, loaded-commit checks, webview change ID parsing, and cached changed-file lookup.
- Route `ShowLogViewProvider` file actions, commit hash copy, GitHub open, and commit detail guards through the lookup helper.
- Remove provider-local changed-file lookup and loaded-commit helper logic.
- Preserve existing Show Log file action behavior, cached changed-file indexing, commit guards, and invalid message no-op behavior.
- Add focused tests for hidden/visible repository lookup, loaded commit checks, cached change lookup, and change ID parsing.

## Acceptance Criteria

- Hidden Show Log state does not expose a repository or changed files.
- Loaded commit checks still accept only commits currently present in the visible log state.
- Cached changed files still resolve from webview IDs shaped as `<commitHash>:<index>`.
- Invalid or stale commit/change IDs still resolve to no action.
- Focused Show Log state lookup, shared-state, message handler, and webview tests pass.
- `npm run build` passes.
- `npm test` passes.
- `git diff --check` passes.

## Verification

- Focused Show Log state lookup, shared-state, message handler, and webview tests passed with 14 tests after extracting state lookup. This includes `npm run build` through the focused validation command.
- `npm test` passed with 339 tests after extracting Show Log state lookup. This includes `npm run build` through the test script.
- `git diff --check` passed after extracting Show Log state lookup and updating verification artifacts.

## Risks

- File actions can regress if webview change ID parsing or cached-change indexing drifts from the webview state payload.
- Runtime integration still needs manual Extension Development Host smoke testing because the helper is exercised by live webview file and commit actions.
