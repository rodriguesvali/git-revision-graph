# 0.0.37 Show Log Commit Graph Polish

## Goal

Improve Show Log compact graph readability against VS Code Source Control by distinguishing merge commits from normal commits and preventing the top-row lane from visually continuing above the first visible commit.

## Scope

- Add Show Log webview commit metadata for merge commits.
- Render Show Log merge commits with a larger double-ring marker, while normal commits keep the existing compact circle.
- Render simple Show Log commits as solid lane-colored circles.
- Stop the first visible Show Log lane at the top commit node instead of drawing a vertical segment above it.
- Add regression coverage for Show Log merge metadata, merge markers, and lane endpoint rendering.

## Non-Goals

- Replacing the layout algorithm.
- Changing commit projection rules.
- Changing the main Revision Graph view.
- Redesigning branch lane colors or Show Log edge routing wholesale.

## Acceptance Criteria

- Commits with multiple parents are identifiable in Show Log.
- Normal Show Log commits use solid lane-colored markers.
- The lane that connects into the top visible Show Log commit terminates at that commit instead of extending above it.
- Build and tests pass.
