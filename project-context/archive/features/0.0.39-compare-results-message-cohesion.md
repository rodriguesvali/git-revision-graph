# Compare Results Message Cohesion

## Goal

Continue the `0.0.39` architectural cohesion work by separating Compare Results webview message validation and dispatch from the editor panel provider lifecycle.

## Scope

- Add `src/compareResults/messageHandler.ts` as the typed owner of Compare Results webview message dispatch.
- Keep `src/compareResults/messageValidation.ts` as the message-shape validation boundary.
- Route `CompareResultsViewProvider` message handling through the dispatcher while preserving item lookup, clipboard, diff, worktree-compare, and restore behavior.
- Preserve all current Compare Results message names, payload shapes, panel behavior, diff operations, and restore confirmation flow.
- Add focused tests for valid and malformed Compare Results message dispatch.

## Acceptance Criteria

- Invalid Compare Results webview messages are ignored before they reach panel actions.
- Ready, base diff, worktree diff, file-name copy, full-path copy, and restore messages route to the same provider actions as before.
- The Compare Results provider retains ownership of panel lifecycle, webview state shaping, item lookup, worktree refresh, and VS Code workbench integrations.
- Focused Compare Results message and webview validation tests pass.
- `npm run build` passes.
- `npm test` passes.
- `git diff --check` passes.

## Verification

- Focused Compare Results message handler, webview message validation, and Compare Results webview tests passed with 11 tests after extracting the dispatcher. This includes `npm run build` through the focused validation command.
- `npm test` passed with 323 tests after extracting Compare Results message dispatch. This includes `npm run build` through the test script.
- `git diff --check` passed after extracting Compare Results message dispatch and updating verification artifacts.

## Risks

- Message routing could drift if a new Compare Results message is added to validation without a dispatcher handler.
- The provider still owns item lookup and worktree-refresh behavior; a future slice may split state shaping or refresh handling if further architectural cohesion is needed.
- Runtime integration still needs manual Extension Development Host smoke testing because message delivery depends on a live VS Code webview.
