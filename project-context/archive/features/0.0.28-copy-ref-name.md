# 0.0.28 Copy Ref Name

## Goal
Let users copy a visible graph reference name directly from the graph context menu.

## User Workflow
1. The user opens the context menu on a visible graph reference.
2. The menu shows `Copy ref name to clipboard`.
3. The user selects the action.
4. The extension copies the selected reference name to the VS Code clipboard.

## Scope
- Add a graph webview context menu action for referenced targets.
- Send a bounded, validated `copy-ref-name` message from the webview to the extension host.
- Restrict the host action to references that are present in the current graph state.
- Use VS Code clipboard APIs and native information feedback.

## Non-Goals
- Adding a Command Palette command.
- Copying synthetic structural commit labels as ref names.
- Changing existing copy commit hash behavior.

## Acceptance Criteria
- Reference context menus include `Copy ref name to clipboard`.
- Structural commit context menus do not expose the ref-name copy action.
- The extension host copies only known visible references.
- Existing graph context menu actions continue to render and validate.

## Manual Validation
- Open the Extension Development Host on a Git repository.
- Right-click a local branch, remote branch, tag, or `HEAD` ref in the graph.
- Select `Copy ref name to clipboard`.
- Paste into an editor and verify the exact ref name was copied.
- Right-click an unreferenced structural commit and verify only commit-oriented inspect actions are shown.
