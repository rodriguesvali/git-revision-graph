# 1.1.0 Stash Ref Actions

## Goal

Expose common stash workflows from the revision graph context menu without adding global toolbar commands.

## Scope

- Add `Stash Save` on the `HEAD` reference context menu only when workspace changes are present and there are no merge conflicts.
- Save workspace changes with the fixed stash label `stash`.
- Add `Stash Apply`, `Stash Pop`, and `Remove Stash` on visible `stash` references.
- Route stash actions through validated webview messages and the existing reference action workflow boundary.
- Use targeted Git CLI calls for `git stash push --include-untracked`, `git stash apply`, `git stash pop`, and `git stash drop`.
- Refresh the graph after stash mutations so stash refs and workspace state update.

## Non-Goals

- No command palette contributions for stash workflows.
- No multi-stash picker or stash list browser beyond the currently visible graph stash ref.
- No change to stash rendering or the `Show Stash` visibility option.

## Acceptance Criteria

- `HEAD` shows `Stash Save` only when the workspace is dirty and not conflicted.
- `Stash Save` uses the fixed stash label `stash`.
- A visible `stash` reference shows apply, pop, and remove actions.
- Apply and pop require a clean workspace before mutating files.
- Remove stash is confirmed and can run even when ordinary workspace changes are present.
- Automated build and test verification pass.
