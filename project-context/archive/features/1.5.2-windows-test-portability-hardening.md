# 1.5.2 Windows Test Portability Hardening

## Status

Implementation complete and verified locally. The maintainer reported on 2026-06-28 that
the Node 20 GitHub Actions matrix passed on Ubuntu, Windows, and macOS. Package metadata is
bumped to `1.5.2`; Marketplace publication was confirmed by the maintainer on 2026-06-28.

## Problem

After the shell-independent compiled test runner began discovering the complete suite on
Windows, `npm test` ran 496 tests but reported 31 failures. The failures formed three
platform-specific clusters rather than independent product regressions:

- repository-relative paths used Windows `\\` separators in logical Git paths, item IDs,
  labels, clipboard selection, unified-diff inputs, and Show Log state;
- fake Git fixtures depended on POSIX shell scripts or direct `.cmd` execution and cleanup
  raced Windows process-tree termination;
- restore fixture content inherited the runner's global `core.autocrlf` setting.

## Goal

Make the existing test and runtime contracts deterministic across Linux, macOS, and Windows
without enabling shell execution or changing user-visible Git workflows.

## Scope and Acceptance Criteria

- Normalize repository-relative logical paths to Git-style `/` separators while preserving
  absolute filesystem paths outside the repository.
- Launch synthetic Git programs as `node <fixture.cjs> ...gitArgs` on every platform.
- Keep `spawn` shell-free so refs and paths are never interpreted by a command shell.
- Await Windows `taskkill` completion before rejecting canceled, timed-out, or output-limited
  Git executions and before test fixtures remove their temporary directories.
- Disable `core.autocrlf` inside content-sensitive Git fixtures.
- Preserve dependency declarations, commands, views, and Marketplace surface.
- Pass `npm run build`, `npm test`, `npm run test:platform`, and `git diff --check` locally.

## Implementation

- `getRepositoryRelativeUriPath` now converts only platform separator characters returned by
  `path.relative` into `/`.
- `configureGitExecutablePath` accepts an internal argument prefix used by shell-free synthetic
  executables; normal extension activation continues to configure only the Git executable path.
- Git abort, timeout, and output-limit rejection now waits for Windows process-tree termination.
- Shared test helper `test/fakeGitExecutable.ts` creates Node-backed fake Git executables.
- Revision graph backend fixtures were converted from POSIX shell programs to CommonJS programs.
- Restore and shared Git fixtures configure repository-local `core.autocrlf=false`.

## Verification Results

- Focused regression set: 67 passed, 0 failed.
- `npm run build`: passed.
- `npm test`: 496 passed, 0 failed.
- Compiled suite with Node `20.19.5`: 496 passed, 0 failed.
- `npm run test:platform`: 29 passed, 0 failed.
- `git diff --check`: passed before artifact updates and remains a final handoff check.
- GitHub Actions Node 20 matrix: passed on Ubuntu, Windows, and macOS by maintainer report

- Post-bump local verification on 2026-06-28: `npm run build`, `npm test`,
  `npm run test:platform`, and `git diff --check` passed.

## Risks and Rollback

- The runtime process change affects only rejection timing after forced termination; successful
  Git commands and normal non-zero exits keep their existing behavior.
- The executable argument prefix is an internal injection seam used by tests and is empty for
  normal extension activation.
- Roll back the process-wait change independently if Windows termination stalls, while retaining
  the shell-free fake executable and path normalization fixes.
