# Render Request Stale Guard

## Goal

Prevent obsolete asynchronous revision graph refreshes from applying state or snapshot side effects after a newer refresh supersedes them.

## Implementation

- Capture an immutable render request context with the coordinator request id and refresh intent for each scheduled build.
- Use the captured intent inside `buildNextState` instead of reading the mutable `latestRefreshIntent` during async work.
- Return `undefined` when a build observes that its request id is no longer current after metadata patching, projection rebuild, or full snapshot loading.
- Apply `currentSnapshot` and mark the snapshot reload complete only for the current render request.
- Keep `RevisionGraphRenderCoordinator` as the final arbiter of applied, discarded, and failed render outcomes.
