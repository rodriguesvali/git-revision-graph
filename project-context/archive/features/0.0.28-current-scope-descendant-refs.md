# Current Scope Descendant References

## Build Status

Build implementation complete for the `0.0.28` current-scope refinement.

Version bump, package generation, and Marketplace publishing remain unapproved.

## Goal

Let users opt into showing references whose tips descend from the current `HEAD` when the graph scope is `Current Branch`.

## User Workflow

A user is on `main`, creates or fetches a branch based on the current `HEAD`, switches the graph scope to `Current Branch`, and enables `Show Current Branch Descendants` from the `View` menu. The graph should show that descendant reference so the user can see work growing from the current branch.

## Scope

- Add a `View` menu toggle named `Show Current Branch Descendants` that is available only when scope is `Current Branch`.
- Keep the toggle disabled by default in projection options.
- Load all refs for the `Current Branch` graph source only when the descendant toggle is enabled.
- Project the current scope as `HEAD` ancestry plus visible references reachable by following children from `HEAD` only when enabled.
- Keep unrelated sibling branches outside the current scope.
- Preserve existing branch/tag/remote/stash visibility toggles.
- Clear `Show Current Branch Descendants` whenever the scope changes away from `Current Branch`.

## Non-Goals

- Changing the default graph scope.
- Adding a new scope selector option.
- Full-history rendering beyond the existing bounded window.
- Changing compare, checkout, merge, or delete workflows.

## Acceptance Criteria

- `Current Branch` still includes the current `HEAD` ancestry.
- `Current Branch` includes visible refs whose tip commits descend from the current `HEAD` when `Show Current Branch Descendants` is enabled.
- `Current Branch` hides descendant refs by default.
- `Current Branch` does not include unrelated sibling branch refs.
- Current-scope Git log arguments load all refs only when descendant refs can be projected.
- The descendant toggle is hidden/disabled outside `Current Branch`, and projection options normalize it to `false` for other scopes.
- Automated tests cover Git args, projection behavior, refresh invalidation, and webview message/state compatibility.

## Manual Validation

- In a real repository, create a branch from the current `HEAD`, add a commit on that branch, return to the original branch, select `Current Branch`, and enable `Show Current Branch Descendants`.
- Confirm the descendant branch reference appears.
- Disable `Show Current Branch Descendants` and confirm the descendant branch reference is hidden again.
- Switch to `All Refs` or `Local Branches` and confirm the descendant option is not available.
- Confirm an unrelated sibling branch from an older base does not appear.

## Release-Note Impact

Mention as a graph-scope refinement: the `View` menu can now show refs that descend from the current `HEAD` while using `Current Branch`.

## Handoff Notes

- Current-scope graph loading still uses `HEAD` by default and switches to `--all` only when descendant refs are enabled; the projection filters the visible graph.
- The controller normalizes descendant visibility to `false` when the scope is not `Current Branch`, so stale webview messages cannot keep it active.
