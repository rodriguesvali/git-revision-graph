# 0.0.37 Show Log Hover Tooltip

## Goal

Add a Source Control-style hover tooltip to Show Log commit rows so users can inspect commit details without expanding the row or opening commit details.

## Scope

- Carry the full commit message into the Show Log webview state.
- Render a hover tooltip for commit rows using loaded Show Log data.
- Include author, date, refs, subject, body, co-author footers, stats, and short hash when available.
- Position the tooltip near the mouse cursor and keep it inside the viewport.
- Wait 0.5 seconds of sustained commit-row hover before opening the tooltip.
- Keep the tooltip interactive while the mouse moves from the row to the tooltip.
- Provide tooltip actions to copy the loaded commit hash and open the commit on GitHub when a GitHub remote is configured.
- Hide the tooltip during clicks, context menus, file-row actions, state changes, and when leaving the commit list.
- Add focused Show Log state, webview shell, message validation, and GitHub URL coverage.

## Non-Goals

- Changing the main Revision Graph view.
- Loading remote avatars.
- Adding new Git commands or extension-host hover providers.

## Acceptance Criteria

- Hovering a Show Log commit row displays a styled commit tooltip.
- The tooltip opens only after a 0.5-second hover delay and is canceled when the cursor leaves the commit row.
- Moving the mouse from a commit row into the tooltip keeps the tooltip available for action clicks.
- Hovering changed-file rows does not show the commit tooltip.
- Clicking `Copy Hash` copies the selected loaded commit hash.
- Clicking `Open on GitHub` opens the commit URL for configured GitHub remotes and reports when no GitHub remote is available.
- The tooltip uses the current Show Log state and does not require extra Git calls.
- Build and tests pass.
