# 1.5.0 Modal Merge Conflict Dialog

## Status

Implementation and verification complete. Automated checks passed and the user reported
the final smoke test complete on 2026-06-28.

## Problem

When a merge started by Git Revision Graph leaves conflicts, the extension awaits a
non-modal VS Code error notification. The repository mutation lease therefore remains
active until the notification is dismissed, while the notification itself is easy to
miss. Attempts to start another operation are rejected as overlapping work and add a
second notification that obscures the original conflict.

## Goal

Make the merge-conflict handoff explicit by showing the extension-owned merge error as
a native modal VS Code dialog while preserving the per-repository mutation guard.

## Scope

- Add VS Code-style modal/detail options to the testable ref-action error UI contract.
- Show merge errors modally only when the error or repository state indicates unresolved
  workspace conflicts.
- Refresh the graph conflict state before waiting for the dialog to close.
- Keep non-conflict errors and all unrelated notification paths unchanged.
- Preserve the existing Source Control handoff and `Abort Merge` workflow.

## Acceptance Criteria

- A merge conflict produces a native modal error dialog instead of a corner notification.
- The mutation lease remains active until the user dismisses the modal dialog.
- Conflict state refresh is requested before the dialog is dismissed.
- After dismissal, the mutation lease is released and the existing conflict guards apply.
- Operations in other repositories retain the coordinator's existing independent behavior.
- Non-conflict merge errors remain non-modal.

## Risks

- Native modal messages block the full VS Code workbench, not only the affected repository.
  The modal path is therefore restricted to unresolved merge conflicts.
- Changing the generic error adapter globally would make unrelated failures modal; the new
  options remain opt-in to avoid that regression.

## Verification Plan

## Implementation Notes

- `RefActionUi.showErrorMessage` now accepts the framework-neutral subset of VS Code
  message options used by the extension: `modal` and `detail`.
- `createWorkbenchRefActionServices` forwards those options to the native
  `vscode.window.showErrorMessage` API.
- `mergeResolvedReference` detects conflict-producing failures, requests the conflict
  state refresh, and then awaits a modal error dialog. Non-conflict merge failures keep
  the existing non-modal call.
- The modal detail directs the user to resolve conflicts in Source Control or abort the
  merge from the `HEAD` reference.
