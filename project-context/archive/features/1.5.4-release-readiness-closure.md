# 1.5.4 Release Readiness Closure

## Goal

Close the `1.5.4` implementation scope and prepare accurate release artifacts for
maintainer-controlled VSIX packaging and Marketplace publication.

## Context

The published baseline is `1.5.3`. Version metadata already reports `1.5.4`, and the completed
candidate combines the remote permission modal-alert correction with Focus Range graph
navigation and its subsequent UX refinements.

## Scope

- Confirm `package.json`, `package-lock.json`, and root lockfile metadata remain on `1.5.4`.
- Freeze the candidate to remote permission modal alerts and Focus Range.
- Align `CHANGELOG.md`, release prioritization, deployment, operations, and release-readiness
  notes with the final implementation.
- Record final deterministic verification and outstanding human release gates.
- Record graph-driven branching workflow profiles as deferred to `2.0.0`.

## Non-Goals

- No VSIX packaging or Marketplace publication in this artifact-only step.
- No version bump beyond `1.5.4`.
- No removal of pull, push, sync, or fetch actions while their product ownership remains under
  review.
- No Git Flow or generic workflow-profile implementation.

## Acceptance Criteria

- Release artifacts describe both implemented `1.5.4` outcomes without expanding scope.
- Package and lockfile versions agree on `1.5.4`.
- Final automated verification is recorded with the current test count.
- Completed automated checks, maintainer-reviewed Focus Range behavior, pending remote-error
  smoke validation, packaging, and publication are reported separately.
