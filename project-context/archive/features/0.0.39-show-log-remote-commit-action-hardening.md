# 0.0.39 Show Log Remote Commit Action Hardening

## Goal

Continue the architectural bad-practices hardening work by moving the Show Log "Open Commit on GitHub" action policy out of the editor panel provider.

## Problem

`ShowLogViewProvider` still owned the external navigation policy for opening a loaded commit on GitHub: building the GitHub URL, deciding the fallback message when no GitHub remote exists, and calling VS Code external-open APIs. That mixed panel lifecycle and loaded-state lookup with remote action behavior.

## Scope

- Added `src/showLog/remoteCommitAction.ts` as the owner for opening a Show Log commit on GitHub.
- Kept `src/showLog/remoteCommitUrl.ts` as the pure URL builder.
- Routed `ShowLogViewProvider` through `openShowLogCommitOnGitHub` after repository and loaded-commit validation.
- Preserved existing behavior for:
  - preferring the existing GitHub URL builder;
  - showing `No GitHub remote is configured for this repository.` when no GitHub remote URL can be built;
  - opening the external GitHub commit URL when available.
- Added focused tests for successful external open and no-GitHub-remote fallback.

## Acceptance Criteria

- `ShowLogViewProvider` no longer owns remote commit URL opening or fallback message behavior.
- Remote commit action behavior is testable without loading VS Code extension-host APIs.
- Existing Show Log message dispatch, loaded-state lookup, remote URL parsing, and rendering behavior remain unchanged.
- Focused Show Log validation and full automated verification pass.

## Verification

- `npm run clean:test && npm run build && tsc -p ./tsconfig.test.json && node --test out-test/test/showLogRemoteCommitAction.test.js out-test/test/showLogRemoteCommitUrl.test.js out-test/test/showLogStateLookup.test.js out-test/test/showLogMessageHandler.test.js out-test/test/showLogWebview.test.js` passed with 13 tests.

## Risks

- Default services lazily load VS Code APIs so pure tests do not require the extension host.
- Manual Extension Development Host smoke testing should still confirm the Show Log commit context action opens the expected GitHub commit page.
