# 1.2.0 Nonblocking Current Branch Sync Error

## Status

Implemented for the `1.2.0` cycle.

## Objective

Ensure the revision graph finishes the `Synchronizing current branch...` action state when Git sync fails, without waiting for the user to close the displayed error message.

## Scope

- Keep the existing sync behavior through the VS Code Git repository API.
- Keep the current error message content, including Git stderr and exit code details.
- Change only the error-notification waiting behavior for `syncCurrentHeadWithUpstream`.
- Preserve conflict handling that can reveal Source Control after the error notification path.

## Implementation Notes

- `syncCurrentHeadWithUpstream` now starts `services.ui.showErrorMessage(...)` and returns `false` without awaiting the notification dismissal.
- If the error indicates unresolved workspace conflicts, Source Control reveal remains chained after the notification promise, matching the existing pull-current-branch behavior.
- The prepared full-rebuild refresh is still canceled before the error is reported, so failed sync attempts do not schedule a stale graph reload.
