# 1.2.0 Deterministic Out Cleanup

## Goal

Ensure every production build starts from an empty `out/` directory so removed or renamed TypeScript modules cannot remain in compiled output or enter a later VSIX.

## Context

The TypeScript compiler writes into `out/` but does not delete files that no longer have matching sources. Before this slice:

- `out/` contained 256 files and was approximately 1.8 MB;
- 11 compiled JavaScript files had no matching current TypeScript source;
- the recently removed `src/viewLayout.ts` still existed as `out/viewLayout.js`;
- older removed graph and tree implementations also remained in compiled output.

`out/` is ignored by Git, so stale files are easy to miss during review while still being visible to `vsce package`.

## Scope

- Add a cross-platform `clean:out` npm script using Node's `fs.rmSync`.
- Add an npm `prebuild` lifecycle script that runs `clean:out`.
- Keep the existing `build` command focused on `tsc -p ./`.
- Verify a clean build contains only JavaScript generated from current TypeScript sources.
- Update `1.2.0` release artifacts.

## Non-Goals

- No source-map exclusion or `.vscodeignore` changes.
- No bundling, minification, dependency changes, or VSIX packaging.
- No orphan-output regression test in this slice; automated package-output enforcement remains a separately prioritized item.
- No change to the long-running `watch` command.

## Touched Files

- `package.json`
- `CHANGELOG.md`

## Acceptance Criteria

- `npm run build` automatically removes the existing `out/` directory before invoking TypeScript.
- Removed and renamed source modules do not survive a clean build.
- The compiled layout worker remains present after the clean build.
- `npm run build`, `npm test`, and `git diff --check` pass.

## Test Plan

- Record pre-change file count, size, and orphaned JavaScript list.
- Run `npm run build`.
- Compare generated `.js` paths against current `.ts` source paths.
- Confirm `out/revisionGraph/layout/d3DagSugiyamaLayoutWorker.js` exists.
- Run `npm test`.
- Run `git diff --check`.

## Manual Validation

- No product workflow changes require a dedicated Extension Development Host smoke path.
- Packaging validation remains part of the later source-map/package-hygiene slice and requires explicit approval before generating a VSIX.

## Risks

- Cleaning `out/` makes an interrupted or failed build leave no runnable previous output. This is intentional: stale runtime artifacts are less trustworthy than an explicit build failure.
- `npm run watch` does not use the `prebuild` lifecycle. Developers switching branches during a watch session should restart it or run a normal build before packaging.

## Verification

- Pre-change `out/` baseline:
  - 256 files;
  - approximately 1.8 MB;
  - 11 compiled JavaScript files without matching current TypeScript sources.
- `npm run build` executed `prebuild`, then `clean:out`, then TypeScript compilation.
- Post-build `out/` result:
  - 234 files;
  - approximately 1.6 MB;
  - zero compiled JavaScript files without matching current TypeScript sources.
- `out/revisionGraph/layout/d3DagSugiyamaLayoutWorker.js` remained present.
- `npm run build` passed.
- `npm test` passed with 446 tests and exercised the clean-build lifecycle.
- `git diff --check` passed.

## Release-Note Impact

- Production builds now clear stale compiled files before TypeScript compilation.
