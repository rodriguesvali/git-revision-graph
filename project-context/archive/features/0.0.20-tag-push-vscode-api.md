# 0.0.20 Tag Push via VS Code Git API

## Goal
Fix tag publishing from the Revision Graph so `Push Tag to Remote` uses the built-in VS Code Git repository API instead of a direct non-interactive Git CLI push.

## Scope
- Keep the release focused on this correction only.
- Push the selected local tag with an explicit tag refspec: `refs/tags/<tag>:refs/tags/<tag>`.
- Preserve remote selection and explicit confirmation before pushing.
- Preserve targeted Git CLI usage for flows that still need it, including local tag creation, remote tag deletion, remote discovery, and graph/history loading.
- Improve the authentication failure message for non-interactive Git credential errors and reveal Source Control as the recovery path.

## Out of Scope
- Version bump, VSIX packaging, or Marketplace publishing without separate human approval.
- Broad branch push or sync behavior changes.
- Replacing graph/history Git CLI calls.
- Pushing all tags from the graph action.

## Acceptance Criteria
- A user can publish the selected local tag through the Revision Graph action without the extension directly running `git push <remote> <tag>`.
- The push uses a fully qualified tag refspec so a branch with the same name is not pushed accidentally.
- If authentication still fails because Git cannot prompt, the user sees a recovery-oriented message that names Source Control and `Git: Push Tags`.
- Existing tag creation, remote tag deletion, branch, compare, checkout, merge, sync, diff, log, and graph behavior remain unchanged.

## Verification Plan
- `npm run build`
- `npm test`
- Manual Extension Development Host smoke:
  - Create or select a local tag.
  - Choose `Push Tag to Remote` from the Revision Graph context menu.
  - Confirm the selected tag appears on the remote.
  - Validate the failure path in an HTTPS remote without CLI credentials if possible.

## Implementation Notes
- `src/workbenchRefActionServices.ts` now calls `repository.push(remoteName, refspec, false)` for tag push.
- `src/refActions/tagRefspec.ts` centralizes tag refspec construction.
- `src/refActions.ts` detects non-interactive Git authentication errors and opens Source Control after showing a focused message.
