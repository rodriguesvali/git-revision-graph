# 1.2.0 Release Opening

## Goal

Open the `1.2.0` AAMAD cycle as a performance-focused minor release after the `1.1.0` implementation baseline.

## Context

The extension already includes substantial graph performance architecture: bounded history loading, cancellation, worker-thread layout, layout caching, load tracing, stale-render guards, and virtualized graph DOM rendering.

The opening analysis identified the next performance ceiling in activation/package cost, repeated graph-load work, Git process fan-out, full-scene viewport scans, and full Show Log rerenders.

## Scope

- Bump `package.json` and `package-lock.json` to `1.2.0`.
- Add the `1.2.0` release prioritization artifact.
- Add the `1.2.0` Define section and architecture decisions.
- Open `1.2.0` release readiness.
- Add a `1.2.0` changelog section without claiming implementation.
- Record baseline findings and unresolved measurement/approval questions.

## Non-Goals

- No runtime performance implementation in the opening step.
- No dependency changes or bundler introduction.
- No command, contribution point, webview, or Git workflow behavior change.
- No VSIX packaging or Marketplace publication.
- No claim that `1.1.0` publication is complete without maintainer confirmation.

## Acceptance Criteria

- Package metadata reports `1.2.0` in both package files.
- Define, Build, and Deliver artifacts identify the performance direction and initial sequencing.
- The release remains explicitly open, unpackaged, and unpublished.
- Candidate optimizations have correctness constraints and measurable verification expectations.
- `git diff --check`, `npm run build`, and `npm test` pass.

## Risks

- A broad performance backlog can overfill a minor release. Use measured slices and defer lower-value work.
- Package and activation optimization can affect worker loading or legacy initialization if implemented without focused validation.
- Current unrelated devcontainer and README edits must remain untouched.

## Verification

- Package metadata confirmed at `1.2.0` in `package.json`, root `package-lock.json`, and root package-lock package metadata.
- `git diff --check` passed.
- `npm run build` passed.
- `npm test` passed with 447 tests.

## Release-Note Impact

- Opening-only note: `1.2.0` is established as the next performance-focused release.
