# 0.0.39 Git Error Test Factory Code Quality

## Goal

Keep Git-like error fixtures in one test helper instead of rebuilding the same object shape in each ref action test.

## Problem

`test/refActions.test.ts` repeated `Object.assign(new Error('Failed to execute git'), ...)` for failures with `stderr`, `gitErrorCode`, and `exitCode`. That made the Git-like error shape noisy in tests and easy to drift from the production error helpers.

## Scope

- Added `TestGitError` and `createGitError` in `test/fakes.ts`.
- Reused `createGitError` across ref action failure scenarios.
- Preserved existing stderr, Git error code, and exit-code assertions.

## Acceptance Criteria

- Ref action tests use the shared Git error fixture helper for Git-like failures.
- Error details and source-control fallback assertions remain unchanged.
- Focused ref action validation and full automated verification pass.

## Verification

- `npm run build` passed after sharing Git-like error fixtures.
- `npm run clean:test && npm run build && tsc -p ./tsconfig.test.json && node --test out-test/test/refActions.test.js out-test/test/errorDetail.test.js` passed with 88 tests.
- `npm test` passed with 394 tests. This includes `npm run build` through the test script.
- `git diff --check` passed after the Git error test factory cleanup and verification artifact updates.

## Risks

- This is a low-risk test-only cleanup. Runtime ref action behavior is unchanged.
