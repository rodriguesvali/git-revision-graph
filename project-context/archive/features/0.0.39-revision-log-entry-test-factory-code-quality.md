# 0.0.39 Revision Log Entry Test Factory Code Quality

## Goal

Reduce duplicated Show Log test setup by sharing a focused `RevisionLogEntry` factory.

## Problem

Multiple Show Log tests recreated the same `RevisionLogEntry` object shape locally. That duplication made routine type changes noisy and increased the chance of slightly different defaults across tests that only care about hashes, labels, parents, or subjects.

## Scope

- Added `createRevisionLogEntry` to `test/fakes.ts`.
- Reused the shared factory from Show Log commit compare, clipboard, reset, lane, and file action tests.
- Removed local `RevisionLogEntry` fixture builders from those tests.
- Preserved test intent by keeping scenario-specific fields explicit at the call sites.

## Acceptance Criteria

- Show Log tests no longer repeat the full `RevisionLogEntry` object shape for common fixtures.
- Existing assertions around commit labels, parent fallback, lanes, clipboard, and compare behavior remain unchanged.
- Focused Show Log validation and full automated verification pass.
