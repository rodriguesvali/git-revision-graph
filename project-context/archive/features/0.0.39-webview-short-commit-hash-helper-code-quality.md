# 0.0.39 Webview Short Commit Hash Helper Code Quality

## Goal

Keep revision graph webview short commit hash display formatting behind one runtime helper.

## Problem

The revision graph webview runtime still had several direct `slice(0, 8)` calls for structural commit summaries, commit selection labels, and search candidates. Host-side TypeScript already uses a shared short-hash helper, so the browser runtime had the same convention duplicated locally.

## Scope

- Added `formatShortCommitHash` to the revision graph webview runtime bootstrap script.
- Reused the helper for structural node summaries and commit selection labels.
- Reused the helper in graph search candidates.
- Added shell-rendering coverage to ensure direct `hash.slice(0, 8)` and `node.hash.slice(0, 8)` calls do not return.

## Acceptance Criteria

- Revision graph webview runtime uses one helper for 8-character commit hash labels.
- Existing search, structural node, and commit action behavior remains unchanged.
- Focused revision graph webview validation and full automated verification pass.
