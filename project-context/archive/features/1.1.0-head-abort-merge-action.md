# 1.1.0 HEAD Abort Merge Action

## Goal

Move conflicted merge abort from the persistent graph toolbar into the `HEAD` reference context menu.

## Context

`Abort Merge` is destructive and was visually prominent in the graph toolbar whenever a conflicted merge was active. The workflow already has a native confirmation, but the toolbar placement made the action feel too aggressive for routine graph browsing.

## Scope

- Remove the visible `Abort Merge` toolbar button from the revision graph shell.
- Keep the existing `abort-merge` webview message and merge-abort workflow unchanged.
- Show `Abort Merge` as a destructive context-menu item only when the clicked graph target is the `HEAD` reference and the repository has a conflicted merge.
- Update webview shell tests to assert the new placement.

## Non-Goals

- No change to merge-abort confirmation copy or Git execution.
- No change to reset, merge, checkout, sync, pull, push, or conflict detection behavior.
- No new command contribution or package manifest change.

## Acceptance Criteria

- The graph toolbar no longer renders an `Abort Merge` button.
- Right-clicking the `HEAD` reference during a conflicted merge exposes `Abort Merge`.
- The action still posts the existing `abort-merge` message and shows the existing host confirmation before running `git merge --abort`.
- Automated build and test verification pass.

## Verification

- `npm run build` passed after moving the action.
- `npm test` passed with 425 tests after updating the webview shell tests.
