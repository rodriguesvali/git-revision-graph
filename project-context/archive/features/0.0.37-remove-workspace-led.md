# 0.0.37 Remove Workspace Status LED

## Goal

Simplify the revision graph toolbar by removing the workspace clean/dirty LED and its redundant shortcut to open Source Control.

## Scope

- Remove the workspace status LED from the revision graph toolbar.
- Remove the LED styling, browser-side click behavior, and `open-source-control` graph webview message contract.
- Preserve workspace dirty/conflict state used by reset, merge, and other guarded workflows.

## Non-Goals

- Removing Source Control handoffs used by conflict or authentication workflows.
- Changing workspace dirty-state loading or guarded Git operations.
- Changing the conflicted-merge `Abort Merge` action.

## Acceptance Criteria

- The revision graph toolbar starts with the Scope control and no longer shows a clean/dirty LED.
- The graph webview cannot send the removed `open-source-control` message.
- Dirty-workspace and conflicted-merge guards continue to use the existing workspace state.
- `npm run build`, focused tests, and `npm test` pass.
