# 0.0.24 Minimap Zoom Levels

## Goal
Give users more minimap zoom range when navigating tall or dense revision graphs.

## User Workflow
Users inspect a large repository graph with the minimap and need more precision than the previous maximum zoom level allowed.

## Scope
- Preserve the existing minimap controls and interaction model.
- Keep the current lower zoom levels.
- Add 10 additional zoom-in levels above the previous maximum.
- Update webview shell regression coverage.

## Non-Goals
- Redesigning the minimap UI.
- Changing graph zoom levels.
- Changing minimap placement, scrolling, or drag navigation.

## Touched Surfaces
- `src/revisionGraph/webview/script/bootstrap.ts`
- `test/revisionGraphWebview.test.ts`

## Acceptance Criteria
- Minimap zoom levels include the previous values.
- The maximum minimap zoom increases from `4` to `30`.
- Zoom-in and zoom-out buttons continue to use the same level stepping logic.

## Test Plan
- Run `npm run build`.
- Run `npm test`.

## Manual Validation
- Open a large revision graph.
- Click the minimap `+` control repeatedly and verify the overview can zoom beyond the old maximum.
- Click the minimap `-` control repeatedly and verify it steps back through the levels.

## Release-Note Impact
- Added 10 more minimap zoom levels for precise navigation in large revision graphs.

## Handoff Notes
- This is a webview runtime constant change. If users still need more precision after validation, consider exposing minimap zoom through a setting rather than extending the fixed array indefinitely.
