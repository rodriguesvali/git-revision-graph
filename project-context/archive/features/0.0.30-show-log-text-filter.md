# Show Log Text Filter

## Build Status

Build implementation complete for the `0.0.30` Show Log text filter.

Version bump, package generation, and Marketplace publishing remain unapproved.

## Goal

Add a TortoiseGit-like text filter to Show Log so users can search commits within the active log source by typing a free-text query.

## Implementation Summary

- Added `filterText` to Show Log extension state and webview state.
- Added a toolbar search field and clear button to the Show Log webview.
- Added a debounced `setFilterText` webview message with bounded validation.
- Reloaded Show Log results when the filter changes, preserving the active source and `Show All Branches` scope.
- Extended backend `loadRevisionLog` with an optional filter argument.
- Implemented case-insensitive matching across commit message, subject, author, hashes, and references.
- Preserved `Load More` behavior for filtered results by paging through matching commits.

## Acceptance Criteria

- Typing a filter updates Show Log results for the active source after a short debounce.
- Matching includes commit description/message, subject, author, hash, and references.
- Clearing the filter restores the unfiltered paged log.
- `Load More` continues to page through filtered results.
- Toggle `Show All Branches` preserves the current filter and reloads the filtered scope.
- Invalid or oversized filter messages are rejected by webview message validation.
- Existing commit expansion and file actions remain scoped to loaded filtered commits.
