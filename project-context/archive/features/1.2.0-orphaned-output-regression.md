# 1.2.0 Orphaned Output Regression Coverage

## Goal

Make stale compiled JavaScript a test failure instead of relying only on manual package inspection.

## Context

The deterministic `prebuild` cleanup removed 11 stale JavaScript files from `out/`, but the invariant was not yet enforced by the automated test suite.

This slice adds direct checks for both parts of the contract:

- production builds must run `clean:out` before TypeScript compilation;
- every generated `out/**/*.js` path must have a matching `src/**/*.ts` path.

## Scope

- Add a package-output test file.
- Lock the `clean:out`, `prebuild`, and `build` script chain.
- Recursively compare compiled JavaScript paths against TypeScript source paths.
- Update `1.2.0` release artifacts.

## Non-Goals

- No package-size threshold or full `vsce ls` invocation inside unit tests.
- No source-map package assertion in this test.
- No bundling, dependency, build-output, or runtime behavior change.
- No VSIX generation or Marketplace publication.

## Acceptance Criteria

- Tests fail if `prebuild` stops invoking `clean:out`.
- Tests fail if a JavaScript file exists under `out/` without the corresponding TypeScript source under `src/`.
- The check supports nested output directories and platform-specific path separators.
- `npm run build`, the focused test, `npm test`, and `git diff --check` pass.

## Test Plan

- Run the new package-output tests after a clean build.
- Run the full test suite.
- Confirm the test suite count increases by two.
- Run `git diff --check`.

## Manual Validation

- None required; this slice adds development-time regression enforcement only.

## Risks

- A future intentional generated JavaScript file without a TypeScript source will need an explicit exception or a revised output contract.
- The test assumes production JavaScript mirrors TypeScript paths, which matches the current `tsc` architecture and should be revisited if bundling is adopted.

## Verification
