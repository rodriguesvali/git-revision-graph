# d3-dag Layout Worker

## Goal

Move the synchronous `d3-dag` Sugiyama layout calculation off the VS Code extension host thread so large graph layout misses do not block repository events, command handling, or webview message processing.

## Implementation

- Kept the layout cache in the extension host and moved only cache-miss calculation to a Node worker thread.
- Added a serializable layout input containing only projected `nodes` and `edges`, avoiding the full `ProjectedGraph` source graph and `Map` fields at the worker boundary.
- Added a worker host that loads the compiled worker from the same `out/revisionGraph/layout` directory used by the extension runtime.
- Propagated `AbortSignal` from ready-state building through scene layout so obsolete refreshes can terminate the worker.
- Preserved the synchronous `calculateD3DagSugiyamaLayout` implementation as a fallback if worker startup or messaging fails.

## Validation

- Passed: `npm run build`
- Passed: `npm test` with 306 passing tests.
