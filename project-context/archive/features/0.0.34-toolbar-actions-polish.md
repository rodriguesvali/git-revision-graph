# Toolbar Actions Polish

## Scope
- Polish the revision graph toolbar after the performance review.
- Keep existing repository actions and message contracts, exposing already-supported workflows through a compact icon slot.

## Change
- Move the workspace clean/dirty indicator to the left edge of the toolbar before Scope.
- Replace text-heavy Reload and Center HEAD controls with icon-only buttons inside a repository action slot.
- Add icon-only toolbar actions for Sync, Pull, Push, and Fetch All alongside Center and Reload.
- Use codicon-style SVG glyphs for the repository action slot so the toolbar matches VS Code Source Control more closely.
- Remove the Remote submenu from the graph context menu now that remote current-branch actions live in the toolbar.
- Ensure no-op or canceled current-branch remote actions post the current graph state back to the webview so toolbar loading feedback is cleared.

## Acceptance Criteria
- The workspace LED appears before the Scope control and keeps its clean/dirty behavior.
- Repository action buttons are icon-only, accessible via labels/tooltips, and disabled when the required repository/upstream state is unavailable.
- The context menu no longer renders a Remote submenu.
- Sync, Pull, and Push clear loading feedback when they finish without scheduling a graph refresh.
- Existing revision graph webview tests remain green.

- `npm run build` passed.
