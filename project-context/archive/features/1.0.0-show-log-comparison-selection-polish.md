# 1.0.0 Show Log Comparison Selection Polish

Build implementation complete for the `1.0.0` Show Log comparison selection polish.

## Goal

Make Show Log commit selections easier to understand when users select two commits for comparison.

## Scope

- Make plain commit clicks expand or collapse the commit without changing the comparison selection.
- Use only `Ctrl` + left mouse click to add, remove, or rotate comparison selections.
- Keep the current two-commit comparison workflow and message contract.
- Improve only the visual treatment of selected commits in the Show Log webview.

## UX Decision

The previous left-edge marker was too weak as the primary selection cue. Selected comparison rows now combine:

- A subtle full-row background tint.
- A thin row outline using the same base color for both selected commits.
- A compact `Base` role badge only on the first selected commit.

Selection is intentionally separated from expansion: plain click remains the lightweight commit expansion gesture, while `Ctrl` + left-click is the only gesture that mutates comparison selection.

## Acceptance Criteria

- The first selected comparison commit is visibly marked as `Base`.
- The second selected comparison commit uses the same selection color without an additional role badge.
- Plain commit clicks expand or collapse commits without selecting or clearing comparison rows.
- `Ctrl` + left-click is the only mouse gesture that changes comparison selection.
- Selection emphasis is readable without relying only on the left vertical marker.
- Row density and Show Log commit scanning remain compact.

## Verification

- Passed: `npm run build`.
- Passed: focused Show Log webview shell test, including the `Ctrl` + left-click selection gesture, plain-click expansion behavior, same-color selections, and single `Base` badge.
- Passed: `npm test` with 398 tests.

## Risk

- Small visual-layout risk in narrow Show Log columns. The role badges are contained inside the message column and should truncate the subject before overlapping author/date columns.
