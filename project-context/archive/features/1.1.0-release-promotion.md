# 1.1.0 Release Promotion

## Goal

Promote the active release line from `1.0.2` to `1.1.0`.

## Context

The cycle opened as `1.0.2` for focused bug fixes. During implementation, the scope grew to include visible functional behavior changes:

- Dirty-worktree branch creation, checkout, and pull now align with VS Code Source Control behavior.
- `Reload Revision Graph` now exposes a long-press `With Empty Cache` action.

These changes are small and compatible, but they are more accurately communicated as a minor release than as a patch-only hotfix.

## Scope

- Bump `package.json` and `package-lock.json` package metadata to `1.1.0`.
- Promote active changelog and release-readiness documentation to `1.1.0`.
- Rename current release build artifacts from `1.0.2` to `1.1.0`.
- Preserve the implementation scope and verification history already recorded for the release.

## Non-Goals

- No additional runtime behavior changes.
- No VSIX packaging or Marketplace publication.
- No dependency changes.

## Acceptance Criteria

- Package metadata reports `1.1.0`.
- Active release artifacts and changelog identify the target release as `1.1.0`.
- Build and whitespace verification pass after the promotion.

## Verification

- `npm run build` passed after promoting package metadata and release artifacts to `1.1.0`.
- `git diff --check` passed after the promotion updates.
