# Cache Stabilization

## Build Status

Build implementation complete for the approved low-risk cache stabilization slice.

This artifact extends the `0.0.27` stabilization release scope without adding a new product surface. Version bump, package generation, and Marketplace publishing remain unapproved.

## Goal

Improve cache observability and bounded cache behavior while preserving repository freshness, cancellation semantics, and deterministic graph layout reuse.

## Scope

- Add trace visibility for graph snapshot cache hits, misses, and bypasses.
- Reuse completed graph snapshot cache entries for cancelable refresh calls without sharing a cancelable in-flight Git process.
- Avoid redundant persisted layout-cache writes when the serialized cache payload has not changed.
- Bound Show Log expanded-change caching with a small LRU-style commit budget.

## Non-Goals

- Persistent commit DAG cache.
- Separating immutable DAG/history data from mutable ref and HEAD overlays.
- Remote tag TTL caching.
- Changing ELK layout cache keys or layout behavior.
- Broad refresh architecture changes.

## Implementation Notes

- `src/revisionGraph/backend.ts` now records `snapshot.cache` trace events with `result=hit`, `result=miss`, or `result=bypass` details.
- Completed snapshot cache entries can satisfy later cancelable graph loads. Pending cache entries are not shared with cancelable callers, so aborting one refresh does not terminate work awaited by another consumer.
- `src/extension.ts` tracks the last serialized projected-layout cache payload and skips workspace-state updates when the payload is unchanged.
- `src/showLogShared.ts` owns `addShowLogCachedChanges`, which caps cached expanded commit changes to `SHOW_LOG_CACHED_CHANGES_MAX_COMMITS`.
- `src/showLogView.ts` uses the shared helper for both newly loaded commit changes and cache-hit recency refreshes.

## Acceptance Criteria

- Snapshot cache behavior is visible in graph load traces.
- A completed snapshot can be reused by a cancelable refresh without starting another Git log command.
- Cancelable callers do not share pending snapshot cache promises whose underlying Git command may be aborted.
- Persisted layout cache writes are skipped when the serialized cache payload is unchanged.
- Show Log expanded-change caching is bounded and refreshes recently used entries.
- Existing graph load, layout cache, Show Log, and refresh behavior remains intact.
