# 0.0.39 Change Target Path Code Quality

## Goal

Remove duplicated rename-target path selection from Compare Results and Show Log review helpers.

## Problem

Several review-panel modules repeated the same `renameUri`-first fallback logic when exposing file names, full paths, and webview item metadata. The duplicated logic was small, but it made renamed-file behavior easier to drift across Compare Results and Show Log surfaces.

## Scope

- Reused `getTargetUri` from `src/changePresentation.ts` in:
  - `src/compareResults/itemActions.ts`;
  - `src/compareResults/viewState.ts`;
  - `src/showLog/fileActions.ts`.
- Removed private target-path helpers that duplicated the shared change presentation rule.
- Preserved existing public helper names and return values for callers, clipboard actions, webview state, and tests.

## Acceptance Criteria

- Rename-target path selection is delegated to the shared change presentation helper.
- Compare Results item paths, Compare Results webview `fullPath`, and Show Log file action paths continue to prefer rename targets.
- Focused path/action/view-state validation and full automated verification pass.
