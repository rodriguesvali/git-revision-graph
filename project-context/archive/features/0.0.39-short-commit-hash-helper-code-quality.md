# 0.0.39 Short Commit Hash Helper Code Quality

## Goal

Keep host-side short commit hash display formatting behind a shared helper.

## Problem

Several TypeScript modules formatted commit labels with local `slice(0, 8)` calls. That duplicated the display-length convention across ref presentation, ref actions, Show Log reset labels, revision graph messages, document fallbacks, and node summary helpers.

## Scope

- Added `formatShortCommitHash` and `SHORT_COMMIT_HASH_LENGTH` in `src/commitHash.ts`.
- Reused the helper across host-side TypeScript commit label formatting.
- Preserved the existing 7-character `shortHash` values parsed from Git log output.
- Added direct helper coverage in `test/commitHash.test.ts`.

## Acceptance Criteria

- Host-side 8-character commit hash labels use the shared helper.
- Existing `RevisionLogEntry.shortHash` parsing behavior remains unchanged.
- Focused hash/ref/show-log/revision-graph validation and full automated verification pass.
