# Show Log Reference Badges

## Build Status

Build implementation complete for the `0.0.30` Show Log reference badge polish.

Version bump, package generation, and Marketplace publishing remain unapproved.

## Goal

Improve reference readability in Show Log by making reference badges slightly larger and aligning their visual colors with the revision graph.

## User Workflow

A user opens Show Log from the revision graph and scans commit references. Local branches, remote branches, `HEAD`, and tags should be visually distinguishable using the same color language as graph nodes.

## Scope

- Increase Show Log reference badge font size from `7.5px` to `8.5px`.
- Preserve reference kind in the Show Log webview state instead of flattening references to strings.
- Render a small colored marker under each reference label.
- Use the same color values as the graph:
  - local branch: green `#19d60f`
  - local `HEAD`: red `#d62828`
  - remote branch: amber `#f6d8a8`
  - tag: yellow `#f7f300`
- Keep existing log layout, commit rows, graph lanes, and file action behavior unchanged.

## Non-Goals

- Changing graph reference colors.
- Changing Show Log data loading, pagination, or Git commands.
- Adding new Show Log commands or context-menu actions.
- Publishing or bumping the extension version.
