# 1.0.0 Polish Opening

## Goal

Open the `1.0.0` cycle as a small polish release after the published `0.0.39` hardening release.

## Context

`0.0.39` is published and the release evidence now records completed Extension Development Host smoke validation, VSIX packaging, package review, and Marketplace publication. The next release should avoid broad product expansion and use the `1.0.0` version to stabilize and polish the existing extension surface.

## Scope

- Bump `package.json` and `package-lock.json` to `1.0.0`.
- Create the `1.0.0` release prioritization artifact.
- Open `1.0.0` release readiness in `project-context/3.deliver/release.md`.
- Add a `1.0.0` changelog placeholder for upcoming polish notes.
- Keep actual polish changes separate and selected one by one.

## Non-Goals

- No VSIX packaging or Marketplace publication in the opening step.
- No new feature implementation in the opening step.
- No new commands, menus, views, settings, dependencies, or contribution points.
- No broad architecture or graph model redesign.
- No change to the documented product limitations.

## Acceptance Criteria

- Version metadata is `1.0.0` in both package files.
- Release readiness clearly states that `1.0.0` is open, not packaged or published.
- The prioritization artifact defines small-polish scope, exclusions, validation, Marketplace notes, and risks.
- `CHANGELOG.md` has a `1.0.0` section ready for selected polish items.
- `git diff --check` passes after the opening updates.

## Risks

- Opening `1.0.0` can imply more scope than intended. Keep the release narrative tied to the current product surface.
- Changelog copy should not claim polish work before the specific changes are selected and implemented.
- Packaging and publication remain separate maintainer-approved actions.

## Verification

- Package metadata confirmed at `1.0.0` in `package.json`, root `package-lock.json`, and root `package-lock.json` package metadata after opening the cycle.
- `git diff --check` passed after opening artifacts.
- `npm run build` passed after opening the `1.0.0` cycle.
- `npm test` passed with 394 tests after opening the `1.0.0` cycle. This includes `npm run build` through the test script.
